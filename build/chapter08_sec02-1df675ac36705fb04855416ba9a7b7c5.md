# 8.2 Wie animiere ich ein Objekt?

Wir haben in Abschnitt 8.1 eine Kugel in eine 3D-Szene gesetzt. Sie bewegt sich
nicht. Aber eine Simulation einer Kugelbahn, in der sich nichts bewegt, ist
wenig hilfreich. Das Herzstück jeder Ursina-Simulation ist die
`update()`-Funktion und die Zeitvariable `time.dt`. Mit ihnen bringen wir
Bewegung in die Szene und legen das Fundament für die physikalische Simulation
in Kapitel 9.

## Was passiert in jedem Frame?

Stellen wir uns vor, wir drehen einen Daumenkino-Film: Jede Seite zeigt ein
leicht verändertes Bild, und wenn wir schnell genug blättern, sehen wir eine
flüssige Bewegung. Ursina arbeitet nach demselben Prinzip. In jedem Frame
berechnet die Engine den aktuellen Zustand der Szene, zeichnet ihn und ruft
dann unsere `update()`-Funktion auf. Wir verändern dort die Positionen und
Eigenschaften der Entities, und im nächsten Frame sieht die Szene entsprechend
anders aus.

*Warum brauchen wir `time.dt` und können nicht einfach eine feste Zahl addieren?*

Wenn wir `entity.x += 0.01` schreiben, bewegt sich das Objekt pro Frame um
0.01 m. Auf einem schnellen Rechner mit 120 fps würde die Kugel jedoch doppelt
so schnell laufen wie auf einem langsamen Rechner mit 60 fps. Mit `time.dt`
multiplizieren wir die Geschwindigkeit mit der tatsächlich verstrichenen Zeit
zwischen zwei Frames. Das Ergebnis ist immer gleich schnell, unabhängig vom
Rechner.

## Wie bewege ich ein Objekt mit konstanter Geschwindigkeit?

Die einfachste Animation: eine Kugel bewegt sich mit konstanter Geschwindigkeit
nach rechts.

```{code-cell} python
:tags: [skip-execution]
from ursina import *

app = Ursina()
window.color = color.white

# Rampe als Hintergrund
Entity(model='cube', color=color.gray,
       scale=Vec3(3.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

# Kugel
kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

# Physikalische Größe
GESCHWINDIGKEIT = 0.5   # m/s in x-Richtung

def update():
    kugel.x += GESCHWINDIGKEIT * time.dt

app.run()
```

Die Kugel bewegt sich mit 0.5 m/s nach rechts. Schließen wir das Fenster, wenn
sie aus dem Bild verschwunden ist.

```{admonition} `global`-Variablen in `update()`
:class: note
Variablen, die wir in `update()` nur lesen (wie `GESCHWINDIGKEIT`), brauchen
kein `global`. Variablen, die wir in `update()` verändern wollen (wie eine
Geschwindigkeit, die sich durch Beschleunigung ändert), müssen mit `global`
deklariert werden. Das sehen wir im nächsten Beispiel.
```

## Wie implementiere ich eine Beschleunigung?

Realistischer ist eine Bewegung, bei der eine Kraft die Geschwindigkeit
verändert. Wir führen eine Geschwindigkeitsvariable ein, die im Euler-Cromer-
Verfahren in jedem Frame aktualisiert wird: erst Geschwindigkeit, dann Position.

```{code-cell} python
:tags: [skip-execution]
from ursina import *
import math

app = Ursina()
window.color = color.white

Entity(model='cube', color=color.gray,
       scale=Vec3(3.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

# Physikalische Größen
G              = 9.81
NEIGUNGSWINKEL = 15.0
A_X            = G * math.sin(math.radians(NEIGUNGSWINKEL))

geschwindigkeit = 0.0   # startet aus dem Stillstand

def update():
    global geschwindigkeit

    # Euler-Cromer-Verfahren: erst Geschwindigkeit, dann Position
    geschwindigkeit += A_X * time.dt
    kugel.x         += geschwindigkeit * time.dt

app.run()
```

````{admonition} Lösung
:class: tip
:class: dropdown
Die Kugel stoppt knapp hinter `x = 1.0`, weil der Zeitschritt `time.dt` nicht
beliebig klein ist. In dem Frame, in dem `kugel.x` erstmals `>= 1.0` wird, hat
die Kugel bereits ein kleines Stück über das Ziel hinausgeschossen. Das ist der
Diskretisierungsfehler der Euler-Methode. Mit kleinerem `time.dt` (also höherer
Framerate) wird der Überschuss kleiner. In Kapitel 9 werden wir diesen Effekt
systematisch untersuchen.
````

## Wie hinterlässt ein Objekt eine Spur?

Ursina hat kein eingebautes Spur-Objekt wie VPython. Wir erzeugen eine Spur,
indem wir in regelmäßigen Abständen kleine Kugeln an der aktuellen Position
der Hauptkugel platzieren.

```{code-cell} python
:tags: [skip-execution]
from ursina import *
import math

app = Ursina()
window.color = color.white

Entity(model='cube', color=color.gray,
       scale=Vec3(3.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

G              = 9.81
NEIGUNGSWINKEL = 15.0
A_X            = G * math.sin(math.radians(NEIGUNGSWINKEL))

geschwindigkeit = 0.0
spur_timer      = 0.0
SPUR_INTERVALL  = 0.05   # alle 0.05 s einen Spurpunkt setzen

def update():
    global geschwindigkeit, spur_timer

    geschwindigkeit += A_X * time.dt
    kugel.x         += geschwindigkeit * time.dt

    # Spurpunkt setzen
    spur_timer += time.dt
    if spur_timer >= SPUR_INTERVALL:
        Entity(model='sphere', color=color.cyan,
               scale=0.02, position=kugel.position)
        spur_timer = 0.0

    if kugel.x >= 1.0:
        geschwindigkeit = 0.0

app.run()
```

Der Trick: Jeder Spurpunkt ist eine eigene Entity, die wir erstellen und dann
nie mehr anfassen. Sie bleiben dauerhaft in der Szene sichtbar.

## Wie zeige ich Text in der Szene an?

Mit `Text` können wir Informationen direkt ins Fenster einblenden:

```{code-cell} python
:tags: [skip-execution]
from ursina import *
import math

app = Ursina()
window.color = color.white

Entity(model='cube', color=color.gray,
       scale=Vec3(3.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

anzeige = Text(
    text     = 't = 0.000 s | v = 0.000 m/s',
    position = (-0.85, 0.45),
    scale    = 1.2,
    color    = color.black
)

G, NEIGUNGSWINKEL = 9.81, 15.0
A_X = G * math.sin(math.radians(NEIGUNGSWINKEL))

geschwindigkeit = 0.0
t_sim           = 0.0

def update():
    global geschwindigkeit, t_sim

    geschwindigkeit  += A_X * time.dt
    kugel.x          += geschwindigkeit * time.dt
    t_sim            += time.dt

    anzeige.text = f't = {t_sim:.3f} s | v = {geschwindigkeit:.3f} m/s'

    if kugel.x >= 1.0:
        geschwindigkeit = 0.0

app.run()
```

## Zusammenfassung und Ausblick

In diesem Abschnitt haben wir die `update()`-Funktion als Herzstück jeder
Ursina-Simulation kennengelernt. Die Variable `time.dt` sorgt dafür, dass
Animationen auf jedem Rechner gleich schnell ablaufen. Im Euler-Cromer-
Verfahren aktualisieren wir erst die Geschwindigkeit, dann die Position. Spurpunkte
erzeugen wir durch das Erstellen kleiner Entities in regelmäßigen Abständen,
und mit `Text` zeigen wir Simulationsdaten in Echtzeit an.

Im nächsten Abschnitt setzen wir alle Bausteine zusammen: Wir bauen
Schritt für Schritt eine vollständige Simulation einer Kugel auf einer
geneigten Rampe auf.
