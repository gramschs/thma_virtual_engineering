````{admonition} Übung 9.1 (✩)
:class: tip
**Kräfte auf der schiefen Ebene zuordnen**

Die folgende Tabelle listet vier Aussagen über ein Objekt auf einer schiefen
Ebene. Entscheiden Sie für jede Aussage, ob sie richtig oder falsch ist, und
korrigieren Sie die falschen Aussagen in einem Satz.

| Nr. | Aussage |
| --- | ------- |
| 1 | Die Normalkraft ist immer gleich groß wie die Gewichtskraft. |
| 2 | Die Haftreibungskraft ist stets kleiner als die Gleitreibungskraft. |
| 3 | Der kritische Winkel hängt nur vom Haftreibungskoeffizienten ab, nicht von der Masse. |
| 4 | Ein Objekt, das einmal gleitet, kann auf derselben Rampe nicht wieder zum Stillstand kommen, wenn der Neigungswinkel konstant bleibt. |
````

````{admonition} Lösung
:class: tip
:class: dropdown
| Nr. | Richtig/Falsch | Korrektur |
| --- | -------------- | --------- |
| 1 | **Falsch** | Die Normalkraft beträgt `F_N = m·g·cos(θ)` und ist nur bei θ = 0° gleich groß wie die Gewichtskraft. Bei geneigter Fläche ist sie kleiner. |
| 2 | **Falsch** | Es gilt stets `μ_H > μ_G`, also ist die maximale Haftreibungskraft größer als die Gleitreibungskraft. Genau deshalb ist es schwerer, ein Objekt in Bewegung zu setzen als es in Bewegung zu halten. |
| 3 | **Richtig** | `θ_krit = arctan(μ_H)` – Masse und Erdbeschleunigung kürzen sich heraus. |
| 4 | **Richtig** | Sobald ein Objekt gleitet, wirkt die (kleinere) Gleitreibung. Damit die Nettokraft null wird, müsste `F_H = F_Gleit` gelten, also `g·sin(θ) = μ_G·g·cos(θ)`, also `tan(θ) = μ_G`. Da `μ_G < μ_H`, liegt dieser Winkel unterhalb des kritischen Winkels – das Objekt kommt nur zum Stillstand, wenn der Neigungswinkel entsprechend klein ist. |
````

````{admonition} Übung 9.2 (✩)

**Kritischen Winkel berechnen und Zustand bestimmen**

Gegeben sind drei Materialpaare mit ihren Reibungskoeffizienten:

| Materialpaar | μ_H | μ_G |
| ------------ | ---- | ---- |
| Stahl auf Stahl | 0.15 | 0.10 |

   Bewegt sie sich? Berechnen Sie, falls ja, die Nettokraft und Beschleunigung.

   zu gleiten beginnen?
````

````{admonition} Lösung
:class: tip
:class: dropdown
```python
import math

materialien = {
    "Stahl auf Stahl": (0.15, 0.10),
    "Gummi auf Beton":  (0.80, 0.60),
    "Holz auf Holz":    (0.40, 0.25),
}

print("Kritische Winkel:")
for name, (mu_h, mu_g) in materialien.items():
    theta_krit = math.degrees(math.atan(mu_h))
    print(f"  {name}: θ_krit = {theta_krit:.1f}°")

# Schwerlastrolle auf Stahl, θ = 12°
print("\nSchwerlastrolle (Stahl, θ=12°):")
m, mu_h, mu_g, theta, G = 5.0, 0.15, 0.10, math.radians(12), 9.81
F_N = m * G * math.cos(theta)
F_H = m * G * math.sin(theta)
F_haft_max = mu_h * F_N
if F_H > F_haft_max:
    F_gleit = mu_g * F_N
    F_netto = F_H - F_gleit
    a = F_netto / m
    print(f"  Gleitet: F_netto = {F_netto:.3f} N, a = {a:.3f} m/s²")
else:
    print(f"  Haftet: F_H={F_H:.3f} N < F_Haft,max={F_haft_max:.3f} N")

# Holzeisenbahn
print(f"\nHolzeisenbahn: θ_krit = {math.degrees(math.atan(0.40)):.1f}°")
```

Ausgabe:
```
Kritische Winkel:
  Stahl auf Stahl: θ_krit = 8.5°
  Gummi auf Beton: θ_krit = 38.7°
  Holz auf Holz:   θ_krit = 21.8°

Schwerlastrolle (Stahl, θ=12°):
  Gleitet: F_netto = 3.481 N, a = 0.696 m/s²

Holzeisenbahn: θ_krit = 21.8°
```

Die Schwerlastrolle gleitet bei 12°, weil der kritische Winkel für Stahl auf
Stahl nur 8.5° beträgt. Die Holzeisenbahn beginnt erst ab ca. 21.8° zu gleiten.
````

````{admonition} Übung 9.3 (✩✩)
**Zustandsübergang systematisch untersuchen**

Implementieren Sie eine Python-Funktion `zustand(m, theta_grad, mu_h, mu_g)`,
die folgendes zurückgibt:

- `"haftet"`, wenn das Objekt stehen bleibt.
- `"gleitet"` sowie die Beschleunigung `a` in m/s², wenn es sich bewegt.

Verwenden Sie die Funktion, um für die Kugelbahn (m = 0.1 kg, μ_H = 0.35,
μ_G = 0.25) eine Tabelle der Zustände und Beschleunigungen für Winkel von
5° bis 40° in 5°-Schritten zu erstellen. Stellen Sie die Beschleunigung als
Funktion des Winkels mit Plotly dar und markieren Sie den kritischen Winkel
als vertikale Linie.
````

````{admonition} Lösung
:class: tip
:class: dropdown
```python
import math
import plotly.express as px
import plotly.graph_objects as go
import pandas as pd

def zustand(m, theta_grad, mu_h, mu_g, G=9.81):
    theta      = math.radians(theta_grad)
    F_N        = m * G * math.cos(theta)
    F_H        = m * G * math.sin(theta)
    F_haft_max = mu_h * F_N
    if F_H <= F_haft_max:
        return "haftet", 0.0
    else:
        F_gleit = mu_g * F_N
        a       = (F_H - F_gleit) / m
        return "gleitet", a

m    = 0.1
MU_H = 0.35
MU_G = 0.25

daten = []
for theta_grad in range(5, 41, 5):
    z, a = zustand(m, theta_grad, MU_H, MU_G)
    daten.append({"Winkel (°)": theta_grad, "Zustand": z, "Beschleunigung (m/s²)": round(a, 4)})
    print(f"θ = {theta_grad:2d}°: {z:8s}  a = {a:.4f} m/s²")

df  = pd.DataFrame(daten)
fig = px.bar(
    df, x="Winkel (°)", y="Beschleunigung (m/s²)", color="Zustand",
    title="Beschleunigung in Abhängigkeit vom Neigungswinkel",
    color_discrete_map={"haftet": "steelblue", "gleitet": "tomato"}
)

theta_krit = math.degrees(math.atan(MU_H))
fig.add_vline(x=theta_krit, line_dash="dash", line_color="gray",
              annotation_text=f"θ_krit = {theta_krit:.1f}°")
fig.show()
```
````

````{admonition} Übung 9.4 (✩✩)
Eine Fahrradkurbel hat folgende Eigenschaften:

3. Stellen Sie Winkelgeschwindigkeit und Winkel als Funktion der Zeit mit
````

````{admonition} Lösung
:class: tip
:class: dropdown
```python
import math
import plotly.express as px
import pandas as pd

# Parameter
I          = 0.003
M_ANTRIEB  = 0.08
M_REIB     = 0.012
M_HAFT     = 0.018
THETA_ZIEL = 6 * math.pi
dt         = 0.005

M_NETTO = M_ANTRIEB - M_REIB

# Anfangszustand
dreht = M_ANTRIEB > M_HAFT
omega = 0.0
theta = 0.0
t     = 0.0

ergebnis = []

while theta < THETA_ZIEL:
    if not dreht and M_ANTRIEB > M_HAFT:
        dreht = True
    alpha  = (M_NETTO / I) if dreht else 0.0
    omega += alpha * dt
    theta += omega * dt
    t     += dt
    ergebnis.append({
        "Zeit (s)": round(t, 4),
        "Winkel (rad)": round(theta, 5),
        "Winkelgeschwindigkeit (rad/s)": round(omega, 4)
    })

df  = pd.DataFrame(ergebnis)
fig = px.line(df, x="Zeit (s)", y="Winkelgeschwindigkeit (rad/s)",
              title="Winkelgeschwindigkeit der Fahrradkurbel")
fig.show()

# Analytischer Vergleich
alpha_analytisch = M_NETTO / I
omega_end_ana    = math.sqrt(2 * alpha_analytisch * THETA_ZIEL)
print(f"Simulation:  ω_end = {omega:.3f} rad/s")
print(f"Analytisch:  ω_end = {omega_end_ana:.3f} rad/s")
print(f"Drehzahl:    {omega / (2*math.pi) * 60:.1f} U/min")
```
````

````{admonition} Übung 9.5 (✩✩✩)
:class: tip
**Einfluss von dt auf die Simulationsgenauigkeit**

Die Euler-Methode ist eine Näherung. Je größer der Zeitschritt `dt`, desto
größer die Abweichung von der exakten Lösung. In dieser Aufgabe untersuchen
wir diesen Zusammenhang systematisch.

Verwenden Sie das translatorische Modell aus Abschnitt 9.2 (θ = 20°,
μ_G = 0.25, μ_H = 0.15, m = 0.1 kg, Bahnlänge 1.2 m) und simulieren Sie
die Rollzeit für folgende Zeitschritte: `dt ∈ {0.1, 0.05, 0.02, 0.01, 0.005,
0.001}`.

1. Berechnen Sie die analytische Rollzeit als Referenz.
2. Berechnen Sie für jeden Zeitschritt die relative Abweichung zur
   analytischen Lösung in Prozent.
3. Stellen Sie die relative Abweichung als Funktion von `dt` auf einer
   logarithmischen x-Achse dar (Plotly).
4. Welchen Zeitschritt würden Sie für eine Genauigkeit unter 0.5 %
   empfehlen?
````

````{admonition} Lösung
:class: tip
:class: dropdown
```python
import math
import plotly.express as px
import pandas as pd

G     = 9.81
THETA = math.radians(20)
MU_H  = 0.15
MU_G  = 0.25
m     = 0.1
s     = 1.2

F_N     = m * G * math.cos(THETA)
F_H     = m * G * math.sin(THETA)
F_GLEIT = MU_G * F_N
a_ref   = (F_H - F_GLEIT) / m   # sofort gleiten, da μ_H = 0.15 < tan(20°)

t_analytisch = math.sqrt(2 * s / a_ref)
print(f"Analytische Rollzeit: {t_analytisch:.5f} s")

dt_werte = [0.1, 0.05, 0.02, 0.01, 0.005, 0.001]
ergebnis = []

for dt in dt_werte:
    v, x, t = 0.0, 0.0, 0.0
    while x < s:
        v += a_ref * dt
        x += v * dt
        t += dt
    abweichung = abs(t - t_analytisch) / t_analytisch * 100
    ergebnis.append({"dt (s)": dt, "Rollzeit (s)": round(t, 5),
                     "Abweichung (%)": round(abweichung, 4)})
    print(f"dt = {dt:.3f} s: t = {t:.5f} s, Abweichung = {abweichung:.4f} %")

df  = pd.DataFrame(ergebnis)
fig = px.line(
    df, x="dt (s)", y="Abweichung (%)", markers=True,
    log_x=True,
    title="Relative Abweichung der Euler-Methode als Funktion von dt",
    labels={"dt (s)": "Zeitschritt dt (s)", "Abweichung (%)": "Relative Abweichung (%)"}
)
fig.add_hline(y=0.5, line_dash="dash", line_color="red",
              annotation_text="0.5 % Grenze")
fig.show()
```

Die Abweichung nimmt proportional zu `dt` ab (Euler-Methode ist ein Verfahren
erster Ordnung). Für eine Genauigkeit unter 0.5 % genügt `dt ≈ 0.01 s`. Für
präzisere Simulationen empfiehlt sich `dt = 0.005 s`, was die Rechenzeit bei
typischen Bahnlängen kaum erhöht.
````

````{admonition} Musterlösung (Kugelbahn)
from vpython import *
import math

scene.title      = "Kugelbahn – Vollständige Simulation"
scene.background = color.white
scene.range      = 0.6

# 1. Parameter (Messung + Literatur)
m              = 0.085          # Masse der Stahlkugel, gemessen
NEIGUNGSWINKEL = 18.0           # Neigungswinkel der Kugelbahn, gemessen
BAHNLAENGE     = 0.95           # Länge der Führungsrille, gemessen
KUGELRADIUS    = 0.0125         # Kugelradius, gemessen

MU_G           = 0.14           # Stahl auf PLA, Literaturwert
G              = 9.81
dt             = 0.005

THETA      = math.radians(NEIGUNGSWINKEL)
F_N        = m * G * math.cos(THETA)
F_H        = m * G * math.sin(THETA)
F_HAFT_MAX = MU_H * F_N
F_GLEIT    = MU_G * F_N

# 2. Kritischer Winkel
theta_krit = math.degrees(math.atan(MU_H))
print(f"Kritischer Winkel: {theta_krit:.1f}° (Bahn: {NEIGUNGSWINKEL}°)")
print(f"Kugel bewegt sich: {F_H > F_HAFT_MAX}")

# Objekte
bahn  = box(pos=vec(0,0,0), size=vec(BAHNLAENGE,0.02,0.05), color=color.gray(0.7))
kugel = sphere(pos=vec(-(BAHNLAENGE/2)+KUGELRADIUS, KUGELRADIUS+0.01, 0),
               radius=KUGELRADIUS, color=color.silver, make_trail=True)
ENDE_X = (BAHNLAENGE/2) - KUGELRADIUS

SKALA     = 2.0
pfeil_FH  = arrow(pos=kugel.pos, axis=vec(F_H*SKALA,0,0),
                  shaftwidth=0.005, color=color.orange)
pfeil_FR  = arrow(pos=kugel.pos, axis=vec(-F_HAFT_MAX*SKALA,0,0),
                  shaftwidth=0.005, color=color.purple)
status    = label(pos=vec(ENDE_X, 0.12, 0), text="", height=11,
                  color=color.black, box=False, align="right")

# Loop
gleitet, v, t = F_H > F_HAFT_MAX, 0.0, 0.0

while kugel.pos.x < ENDE_X:

        p.pos = kugel.pos
    pfeil_FR.axis = vec(-F_R_akt * SKALA, 0, 0)
    status.text   = f"{'Gleiten' if gleitet else 'Haften'}\nt={t:.3f}s\nv={v:.3f}m/s"

# 4. Validierung
a_ref = (F_H - F_GLEIT) / m
t_ana = math.sqrt(2 * BAHNLAENGE / a_ref)
````
