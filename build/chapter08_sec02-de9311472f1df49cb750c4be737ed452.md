## Was ist ein Animations-Loop?

Jede Animation – ob in einem Computerspiel, einem CAD-Tool oder in VPython –
beruht auf demselben Grundprinzip: Wir berechnen die Positionen aller Objekte
für einen kleinen Zeitschritt, zeichnen das Bild neu und wiederholen das so
schnell, dass es dem Auge wie eine flüssige Bewegung erscheint.

In VPython schreiben wir dieses Grundprinzip als `while`-Schleife:

```{code-cell} python
from vpython import *

# Szene einrichten
scene.background = color.white
scene.range      = 0.8

# Objekte erstellen
boden = box(pos=vec(0, -0.01, 0), size=vec(2.0, 0.02, 0.2), color=color.gray(0.6))
kugel = sphere(pos=vec(-0.8, 0.05, 0), radius=0.05, color=color.red)

# Physikalische Größen
GESCHWINDIGKEIT = vec(0.3, 0, 0)   # 0.3 m/s in x-Richtung (Konstante in GROSSBUCHSTABEN)
dt = 0.01                          # Zeitschritt: 10 ms
t  = 0.0                           # Startzeit

# Animations-Loop
while t < 5.0:
    rate(100)                      # maximal 100 Schritte pro Sekunde
    kugel.pos += GESCHWINDIGKEIT * dt
    t += dt
```

Dieser Code bewegt die Kugel mit konstanter Geschwindigkeit von links nach
rechts. Wir schauen uns die drei entscheidenden Bestandteile genauer an.

## Was bewirkt `rate()`?

*Warum können wir die Schleife nicht einfach so schnell wie möglich laufen
lassen?*

Ohne `rate()` würde VPython die Schleife so schnell ausführen, wie der
Prozessor es erlaubt. Auf einem schnellen Rechner könnte das 10 000 Schritte
pro Sekunde sein, auf einem langsamen vielleicht nur 200. Das Ergebnis: Auf
verschiedenen Rechnern läuft die Simulation unterschiedlich schnell. Mit
`rate(100)` begrenzen wir die Zahl der Schleifendurchläufe auf maximal 100 pro
Sekunde und machen die Simulation damit rechnerunabhängig.

Gleichzeitig teilt `rate()` dem Browser oder Jupyter-Kernel etwas Rechenzeit
mit, sodass das 3D-Fenster flüssig aktualisiert werden kann. Ohne `rate()` würde
das Fenster einfrieren, weil die Schleife den gesamten Prozessor blockiert.

```{admonition} Tipp: rate() und Echtzeit
:class: note
`rate(N)` begrenzt die Schleife auf N Durchläufe pro Sekunde. Wenn wir `dt =
0.01` s und `rate(100)` wählen, läuft die simulierte Zeit genauso schnell wie
die echte Zeit: 100 Schritte × 0.01 s = 1 simulierte Sekunde pro echter
Sekunde. Das ist für Demonstrationen schön, aber nicht notwendig. Eine Simulation
kann auch schneller oder langsamer als Echtzeit laufen.
```

## Wie aktualisieren wir Position und Zeit?

Der Kern jedes Zeitschritts sind zwei Zeilen:

```python
kugel.pos += GESCHWINDIGKEIT * dt
t         += dt
```

Die erste Zeile verschiebt die Kugel um den zurückgelegten Weg in einem
Zeitschritt: `Δpos = v · Δt`. Die zweite Zeile rückt die Simulationsuhr vor.

*Wie klein muss `dt` sein?*

Das hängt davon ab, wie stark sich die Bewegung innerhalb eines Zeitschritts
ändert. Bei einer gleichförmigen Bewegung mit konstanter Geschwindigkeit ist
`dt = 0.1 s` ausreichend. Bei einer Bewegung mit schnell wechselnder
Beschleunigung, zum Beispiel auf einer kurvigen Bahn, brauchen wir ein kleineres
`dt`. In Kapitel 9 werden wir auf diesen Kompromiss zwischen Genauigkeit und
Rechenzeit zurückkommen.

## Wie hinterlässt ein Objekt eine Spur?

Um die zurückgelegte Bahn sichtbar zu machen, setzen wir `make_trail = True`
bei der Objektdefinition:

```{code-cell} python
from vpython import *

scene.background = color.white
scene.range      = 0.8

boden = box(pos=vec(0, -0.01, 0), size=vec(2.0, 0.02, 0.2), color=color.gray(0.6))
kugel = sphere(
    pos        = vec(-0.8, 0.05, 0),
    radius     = 0.05,
    color      = color.red,
    make_trail = True               # Spur aktivieren
)

GESCHWINDIGKEIT = vec(0.3, 0, 0)
dt = 0.01
t  = 0.0

while t < 5.0:
    rate(100)
    kugel.pos += GESCHWINDIGKEIT * dt
    t += dt
```

Die Spur zeichnet jeden Punkt nach, den der Mittelpunkt der Kugel durchläuft.
Das ist für die Kugelbahn-Simulation besonders nützlich: Wir sehen auf einen
Blick, ob die berechnete Trajektorie mit der Form der Bahn übereinstimmt.

## Wie implementieren wir eine gleichmäßig beschleunigte Bewegung?

Gleichförmige Bewegung mit konstanter Geschwindigkeit ist der einfachste Fall.
Realistischer ist eine Bewegung, bei der eine Kraft die Geschwindigkeit
verändert. Wir führen dazu ein neues Konzept ein: die Geschwindigkeit als
veränderliche Größe.

In der Physik gilt für jeden kleinen Zeitschritt:

```
Δv = a · Δt
Δpos = v · Δt
```

Zuerst aktualisieren wir die Geschwindigkeit mit der Beschleunigung, dann die
Position mit der (bereits aktualisierten) Geschwindigkeit. Dieser Algorithmus
heißt **Euler-Methode** und ist die einfachste numerische Integrationsmethode:

```{code-cell} python
from vpython import *

scene.background = color.white
scene.range      = 0.6

boden = box(pos=vec(0, -0.01, 0), size=vec(2.0, 0.02, 0.2), color=color.gray(0.6))
kugel = sphere(
    pos        = vec(-0.8, 0.05, 0),
    radius     = 0.05,
    color      = color.red,
    make_trail = True
)

# Physikalische Größen
G              = 9.81               # Erdbeschleunigung in m/s²
NEIGUNGSWINKEL = 10                 # Grad
# Beschleunigungskomponente entlang der geneigten Bahn (vereinfacht):
# a_x = g * sin(θ), die Kugel beschleunigt in x-Richtung
import math
A_X = G * math.sin(math.radians(NEIGUNGSWINKEL))  # ca. 1.7 m/s²

geschwindigkeit = vec(0, 0, 0)      # Startet aus dem Stillstand
dt = 0.01
t  = 0.0

while t < 3.0:
    rate(100)
    beschleunigung   = vec(A_X, 0, 0)
    geschwindigkeit += beschleunigung * dt
    kugel.pos       += geschwindigkeit * dt
    t               += dt
```

## Warum reihenfolgt die Aktualisierung?

Im Euler-Schritt aktualisieren wir erst die Geschwindigkeit, dann die Position.
Würden wir es umgekehrt machen, würden wir die Position mit der noch alten
Geschwindigkeit aktualisieren und die Beschleunigung in diesem Schritt gar nicht
nutzen. Das führt bei großem `dt` zu spürbaren Fehlern. Als einfache Faustregel
gilt:

```python
# Richtige Reihenfolge:
geschwindigkeit += beschleunigung * dt   # erst Geschwindigkeit
kugel.pos       += geschwindigkeit * dt  # dann Position

# Falsche Reihenfolge (führt zu Abweichungen):
kugel.pos       += geschwindigkeit * dt  # alte Geschwindigkeit
geschwindigkeit += beschleunigung * dt
```

In Kapitel 9 werden wir sehen, wie wir die Genauigkeit durch kleineres `dt` und
verbesserte Integrationsverfahren weiter steigern können.

## Zusammenfassung und Ausblick

In diesem Abschnitt haben wir den Animations-Loop als Herzstück jeder
VPython-Simulation kennengelernt. Die Funktion `rate()` sorgt dafür, dass die
Simulation auf jedem Rechner gleich schnell läuft und das 3D-Fenster flüssig
aktualisiert wird. Mit `dt` steuern wir die Zeitauflösung: Kleines `dt` bedeutet
höhere Genauigkeit und mehr Rechenaufwand, großes `dt` spart Zeit, kann aber zu
Fehlern führen. Mit `make_trail = True` hinterlässt die Kugel eine sichtbare
Spur. Und mit der Euler-Methode haben wir unseren ersten physikalischen
Bewegungsalgorithmus implementiert: zuerst Geschwindigkeit aktualisieren, dann
Position.
