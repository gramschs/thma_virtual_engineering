# 8.3 VPython: Erste Schritte und einfache Animation

Wir haben die Theorie: Objekte, Vektoren, Animations-Loop, Euler-Methode. Jetzt
bauen wir alles Schritt für Schritt zusammen. Am Ende dieses Abschnitts haben
wir eine lauffähige VPython-Simulation einer Kugel, die eine geneigte Rampe
hinunterrollt. Das ist noch kein vollständiges physikalisches Modell, aber eine
solide, animierte Szene, in der wir die Grundbausteine in Aktion sehen.

## Schritt 1: Szene einrichten

Jede Simulation beginnt mit der Einrichtung der Szene. Wir legen Hintergrund,
Titel und Kameraausschnitt fest, bevor wir ein einziges physikalisches Objekt
erstellen. Das ist guter Stil und erleichtert später das Anpassen.

```{code-cell} python
:tags: [skip-execution]
from vpython import *
import math

# Szene
scene.title      = "Kugelbahn: einfache Rampe"
scene.background = color.white
scene.range      = 0.7
scene.width      = 900
scene.height     = 500
```

Wir importieren `math` zusätzlich, weil wir `math.sin()` und `math.radians()`
für die Neigungsberechnung benötigen. Der Stern-Import `from vpython import *`
bringt alle VPython-Funktionen direkt in den Namensraum.

## Schritt 2: Rampe und Kugel platzieren

Unsere Rampe ist vereinfacht eine gerade, geneigte Fläche. Wir modellieren sie
als flachen Quader, der um einen Winkel geneigt ist. VPython-Quader können
über ihre `axis`-Eigenschaft gedreht werden. Eine einfachere Alternative ist,
die Neigung über die `up`-Eigenschaft einzustellen. Für unseren Einstieg
verwenden wir eine noch einfachere Näherung: Wir stellen die Rampe horizontal
und beschleunigen die Kugel so, als wäre sie geneigt. Das entkoppelt die
Visualisierung von der Physik, ein Vorgehen, das wir in Kapitel 10 durch ein
geometrisch korrektes Modell ersetzen werden.

```{code-cell} python
:tags: [skip-execution]
from vpython import *
import math

scene.title      = "Kugelbahn: einfache Rampe"
scene.background = color.white
scene.range      = 0.7

# Physikalische Parameter
NEIGUNGSWINKEL = 15.0      # Neigungswinkel der Rampe in Grad
BAHNLAENGE     = 1.2       # Länge der Rampe in Metern
KUGELRADIUS    = 0.04      # Kugelradius in Metern

# Rampe: horizontaler Quader als vereinfachte Darstellung
rampe = box(
    pos   = vec(0, 0, 0),
    size  = vec(BAHNLAENGE, 0.02, 0.12),
    color = color.gray(0.65)
)

# Startposition der Kugel: linker Rand der Rampe, leicht oberhalb
START_X = -(BAHNLAENGE / 2) + KUGELRADIUS
kugel   = sphere(
    pos        = vec(START_X, KUGELRADIUS + 0.01, 0),
    radius     = KUGELRADIUS,
    color      = color.red,
    make_trail = True
)
```

```{admonition} Hinweis: Konstanten in GROSSBUCHSTABEN
:class: note
Physikalische Größen, die sich während der Simulation nicht ändern (Bahnlänge,
Kugelradius, Neigungswinkel), schreiben wir in Großbuchstaben. Das macht den
Code lesbarer: Auf den ersten Blick ist erkennbar, welche Variablen Parameter
der Simulation sind und welche sich im Loop verändern.
```

## Schritt 3: Physikalische Größen initialisieren

Vor dem Loop bereiten wir alle Variablen vor, die sich während der Simulation
verändern werden: Geschwindigkeit, Zeit und Beschleunigung. Ausnahmsweise nehmen
wir für die Erdbeschleunigung g einen Kleinbuchstaben.

```{code-cell} python
:tags: [skip-execution]
# Physikalische Größen (Fortsetzung des vorherigen Codes)
g  = 9.81   # Erdbeschleunigung in m/s²
A_X = g * math.sin(math.radians(NEIGUNGSWINKEL))   # Hangabtriebsbeschleunigung

geschwindigkeit = vec(0, 0, 0)   # Kugel startet aus dem Stillstand
dt = 0.005                       # Zeitschritt in Sekunden
t  = 0.0                         # Simulationszeit
```

Die Hangabtriebsbeschleunigung `A_X = g · sin(θ)` ist die Komponente der
Schwerkraft entlang der Rampe. Bei einem Neigungswinkel von 15° ergibt sich:
`A_X = 9.81 · sin(15°) ≈ 2.54 m/s²`. Das bedeutet: Jede Sekunde nimmt die
Geschwindigkeit der Kugel um etwa 2.5 m/s zu.

```{admonition} Gleiten, nicht Rollen
:class: note
Streng genommen modellieren wir hier eine **gleitende Punktmasse**, keine
rollende Kugel. Die Rotationsdynamik, also das Trägheitsmoment der Kugel
und der Anteil der Energie, der in der Eigenrotation steckt, wird erst
in Kapitel 9 eingeführt. Die hier berechnete "Bewegungszeit" ist daher
kürzer als die reale Rollzeit einer Kugel auf derselben Bahn. In Kapitel 11
werden wir sehen, dass der Unterschied theoretisch dem Faktor √(7/5) ≈ 1.18
entspricht.
```

*Warum wählen wir `dt = 0.005 s` und nicht `dt = 0.1 s`?*

Bei einer Beschleunigung von 2.5 m/s² und `dt = 0.1 s` würde die
Geschwindigkeit pro Schritt um 0.25 m/s springen. Das ist eine grobe Näherung.
Mit `dt = 0.005 s` beträgt der Sprung nur 0.013 m/s und die Simulation ist
deutlich präziser.

## Schritt 4: Anzeige hinzufügen

VPython kann mit dem `label`-Objekt Texte direkt in die 3D-Szene einblenden.
Das ist nützlich, um Simulationszeit und Geschwindigkeit in Echtzeit
anzuzeigen:

```{code-cell} python
:tags: [skip-execution]
# Textanzeige in der Szene (Fortsetzung)
anzeige = label(
    pos     = vec(0.3, 0.15, 0),
    text    = "t = 0.000 s",
    height  = 14,
    color   = color.black,
    box     = False
)
```

Im Animations-Loop aktualisieren wir den Text jedes Mal, wenn wir die Zeit
vorwärtsschieben.

## Schritt 5: Animations-Loop schreiben

Jetzt kommen alle Bausteine zusammen. Der Loop läuft, bis die Kugel das Ende
der Rampe erreicht hat:

```{code-cell} python
:tags: [skip-execution]
from vpython import *
import math

# --- Szene ---
scene.title      = "Kugelbahn – Einfache Rampe"
scene.background = color.white
scene.range      = 0.7

# --- Parameter ---
NEIGUNGSWINKEL = 15.0
BAHNLAENGE     = 1.2
KUGELRADIUS    = 0.04
G              = 9.81
A_X            = G * math.sin(math.radians(NEIGUNGSWINKEL))

# --- Objekte ---
rampe = box(
    pos   = vec(0, 0, 0),
    size  = vec(BAHNLAENGE, 0.02, 0.12),
    color = color.gray(0.65)
)

START_X = -(BAHNLAENGE / 2) + KUGELRADIUS
kugel   = sphere(
    pos        = vec(START_X, KUGELRADIUS + 0.01, 0),
    radius     = KUGELRADIUS,
    color      = color.red,
    make_trail = True
)

anzeige = label(
    pos    = vec(0.3, 0.18, 0),
    text   = "t = 0.000 s | v = 0.00 m/s",
    height = 14,
    color  = color.black,
    box    = False
)

# --- Anfangszustand ---
geschwindigkeit = vec(0, 0, 0)
dt = 0.005
t  = 0.0

# Endposition: rechter Rand der Rampe
ENDE_X = (BAHNLAENGE / 2) - KUGELRADIUS

# --- Animations-Loop ---
while kugel.pos.x < ENDE_X:
    rate(200)

    # Euler-Cromer-Verfahren (symplektischer Euler):
    # erst Geschwindigkeit aktualisieren, dann Position mit neuer Geschwindigkeit
    geschwindigkeit += vec(A_X, 0, 0) * dt
    kugel.pos       += geschwindigkeit * dt
    t               += dt

    # Anzeige aktualisieren
    v_betrag         = mag(geschwindigkeit)
    anzeige.text     = f"t = {t:.3f} s | v = {v_betrag:.2f} m/s"
```

```{admonition} Euler-Cromer-Verfahren
:class: note
Die Reihenfolge "erst Geschwindigkeit, dann Position" ist kein zufälliger
Stilentscheid: Sie entspricht dem **Euler-Cromer-Verfahren** (auch
symplektischer Euler genannt), einer Variante des klassischen expliziten
Euler-Verfahrens. Der Unterschied: Beim klassischen Euler würde die
Position mit der *alten* Geschwindigkeit aktualisiert; hier verwenden wir
die *bereits aktualisierte* Geschwindigkeit. Das Euler-Cromer-Verfahren
ist energieerhaltender und liefert bei Schwingungen und Kreisbewegungen
deutlich bessere Ergebnisse. Für unsere gerade Rampe ist der Unterschied
gering, aber die Gewohnheit, in dieser Reihenfolge zu schreiben, zahlt
sich in Kapitel 9 aus.
```

```{admonition} Checkliste
:class: note
* **Richtung stimmt:** Die Kugel bewegt sich nach rechts (positive x-Richtung).
* **Startposition stimmt:** Die Kugel beginnt am linken Ende der Rampe.
* **Beschleunigung sichtbar:** Die Kugel wird im Verlauf sichtbar schneller.
* **Spur sichtbar:** `make_trail = True` hinterlässt eine rote Linie.
* **Anzeige läuft:** Zeit und Geschwindigkeit werden aktualisiert.
* **Loop endet:** Die Simulation stoppt, wenn die Kugel das rechte Ende
  erreicht.
```

## Schritt 6: Ergebnis prüfen

Nach dem Lauf können wir das Ergebnis analytisch überprüfen. Für eine
gleichmäßig beschleunigte Bewegung gilt:

```code
v_end = sqrt(2 · a · s)
```

Dabei ist `s` die tatsächlich zurückgelegte Strecke des Kugelmittelpunkts –
also nicht die Quader-Länge `BAHNLAENGE`, sondern der Weg von `START_X` bis
`ENDE_X`. Da die Kugel beidseitig um einen Radius eingerückt startet und
stoppt, gilt:

```code
S_EFF = BAHNLAENGE - 2 · KUGELRADIUS
```

```{admonition} Geometriehinweis
:class: note
`BAHNLAENGE` beschreibt die Länge des Quaders (der sichtbaren Rampe).
Der Kugelmittelpunkt bewegt sich jedoch nur über `S_EFF = BAHNLAENGE -
2 · KUGELRADIUS`, weil er weder am linken noch am rechten Rand der Rampe
über die Kante hinausragen darf. Die analytische Rechnung muss diese
effektive Strecke verwenden – sonst vergleichen wir zwei geometrisch
verschiedene Szenarien und erzeugen eine Abweichung, die nichts mit der
Genauigkeit des Euler-Algorithmus zu tun hat.
```

```{code-cell} python
import math

NEIGUNGSWINKEL = 15.0
G              = 9.81
BAHNLAENGE     = 1.2
KUGELRADIUS    = 0.04

# Effektive Wegstrecke des Kugelmittelpunkts
S_EFF = BAHNLAENGE - 2 * KUGELRADIUS

A_X   = G * math.sin(math.radians(NEIGUNGSWINKEL))
V_END = math.sqrt(2 * A_X * S_EFF)
T_END = V_END / A_X

print(f"Quader-Länge BAHNLAENGE:        {BAHNLAENGE:.3f} m")
print(f"Effektive Wegstrecke S_EFF:     {S_EFF:.3f} m")
print(f"Hangabtriebsbeschleunigung:     {A_X:.3f} m/s²")
print(f"Analytische Endgeschwindigkeit: {V_END:.3f} m/s")
print(f"Analytische Bewegungszeit:      {T_END:.3f} s")
```

Vergleichen wir diese Werte mit der Anzeige am Ende der Simulation. Eine kleine
Restabweichung ist normal und hat zwei Quellen: erstens den Diskretisierungsfehler
der Euler-Methode (proportional zu `dt`), zweitens die Tatsache, dass der
Loop abbricht, sobald `kugel.pos.x >= ENDE_X` gilt, was je nach Schrittweite
etwas über das Ziel hinausschießen kann. In Kapitel 9 diskutieren wir beide
Effekte systematisch.

```{admonition} Mini-Übung
:class: tip
Verändern Sie `NEIGUNGSWINKEL` auf 30° und dann auf 5°. Beobachten Sie:

1. Wie verändert sich die Endgeschwindigkeit?
2. Wie verändert sich die Rollzeit?
3. Entsprechen die Änderungen Ihrer Intuition?

Berechnen Sie die analytischen Werte für beide Winkel und vergleichen Sie mit
der Simulation.
```

````{admonition} Lösung
:class: tip
:class: dropdown
```python
import math

G           = 9.81
BAHNLAENGE  = 1.2
KUGELRADIUS = 0.04
S_EFF       = BAHNLAENGE - 2 * KUGELRADIUS   # 1.12 m

for winkel in [5, 15, 30]:
    a     = G * math.sin(math.radians(winkel))
    v_end = math.sqrt(2 * a * S_EFF)
    t_end = v_end / a
    print(f"θ = {winkel:2d}°: a = {a:.2f} m/s², v_end = {v_end:.2f} m/s, t_bew = {t_end:.2f} s")
```

Ausgabe:
```
θ =  5°: a = 0.85 m/s², v_end = 1.38 m/s, t = 1.62 s
θ = 15°: a = 2.54 m/s², v_end = 2.38 m/s, t = 0.94 s
θ = 30°: a = 4.91 m/s², v_end = 3.31 m/s, t = 0.67 s
```

Eine Verdoppelung des Winkels verdoppelt die Beschleunigung nicht, da `sin(θ)`
nicht linear ist. Die Endgeschwindigkeit hängt von der Wurzel der Beschleunigung
ab, die Rollzeit nimmt stärker ab als die Endgeschwindigkeit zunimmt.
````
