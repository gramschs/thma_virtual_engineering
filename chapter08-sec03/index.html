8.3 Ursina: Erste Schritte und einfache Animation

Wir haben die Theorie: Entities, Koordinatensystem, update(), time.dt, Euler-Cromer-Verfahren. Jetzt bauen wir alles Schritt für Schritt zusammen. Am Ende dieses Abschnitts haben wir eine lauffähige Ursina-Simulation einer Kugel, die eine geneigte Rampe hinuntergleitet. Das ist noch kein vollständiges physikalisches Modell (das kommt in Kapitel 9), aber eine solide, animierte Szene, in der wir die Grundbausteine in Aktion sehen.

Lernziele¶

Lernziele

  • Sie können eine strukturierte Ursina-Simulation von Grund auf aufbauen.

  • Sie kennen die Möglichkeiten, eine geneigte Rampe mit Entity über rotation geometrisch korrekt zu positionieren, und können erklären, warum in diesem Abschnitt eine vereinfachte horizontale Darstellung verwendet wird.

  • Sie können die Kugel mit einer konstanten Beschleunigung animieren und eine Textanzeige mit Echtzeit-Daten einblenden.

  • Sie können die Simulation auf Plausibilität prüfen und das Ergebnis mit einer analytischen Lösung vergleichen.

Schritt 1: Szene einrichten¶

Jede Simulation beginnt mit der Einrichtung von Fenster und Hintergrund.

from ursina import *
import math

app = Ursina(title='Kugelbahn: Einfache Rampe')
window.size = Vec2(1200, 700)
window.color = color.white

Schritt 2: Parameter definieren¶

Alle physikalischen Größen stehen am Anfang des Skripts als Konstanten in Großbuchstaben, sodass sie leicht zu finden und anzupassen sind.

Warum lohnt es sich, alle Parameter gesammelt ganz oben zu definieren?

Wer später den Neigungswinkel auf 20° ändern möchte, findet ihn sofort in der ersten Zeile des Parameterblocks, statt ihn in der Schleife suchen zu müssen.

import math

# Physikalische Parameter
NEIGUNGSWINKEL = 15.0       # Grad
BAHNLAENGE     = 1.2        # Länge der Rampe in Metern
KUGELRADIUS    = 0.04       # Kugelradius in Metern
G              = 9.81       # Erdbeschleunigung in m/s²

# Hangabtriebsbeschleunigung (gleitende Punktmasse, ohne Rotation)
A_X = G * math.sin(math.radians(NEIGUNGSWINKEL))

# Effektive Wegstrecke des Kugelmittelpunkts
S_EFF  = BAHNLAENGE - 2 * KUGELRADIUS
START_X = -(BAHNLAENGE / 2) + KUGELRADIUS
ENDE_X  =  (BAHNLAENGE / 2) - KUGELRADIUS

Streng genommen modellieren wir hier eine gleitende Punktmasse, keine rollende Kugel. Die Rotationsdynamik (Trägheitsmoment und Rotationsenergie) wird erst in Kapitel 9 eingeführt. Die hier berechnete Bewegungszeit ist daher kürzer als die reale Rollzeit. In Kapitel 11 werden wir sehen, dass der Unterschied dem Faktor √(7/5) ≈ 1.18 entspricht.

Schritt 3: Rampe und Kugel platzieren¶

In dieser Simulation liegt die Rampe horizontal; die Neigung wird nur in der Physik berücksichtigt. Eine geometrisch korrekte geneigte Rampe lässt sich in Ursina mit rotation_z=-NEIGUNGSWINKEL erzeugen; dann muss aber auch die Kugelposition entsprechend umgerechnet werden. Wir verwenden die vereinfachte horizontale Darstellung, um Visualisierung und Physik bewusst zu entkoppeln. In Kapitel 10 ersetzen wir die Quader-Rampe durch das echte .obj-Mesh.

# Rampe (horizontal, vereinfacht)
rampe = Entity(
    model    = 'cube',
    color    = color.gray,
    scale    = Vec3(BAHNLAENGE, 0.02, 0.12),
    position = Vec3(0, 0, 0)
)

# Kugel
kugel = Entity(
    model    = 'sphere',
    color    = color.red,
    scale    = 2 * KUGELRADIUS,                     # scale = Durchmesser
    position = Vec3(START_X, KUGELRADIUS + 0.01, 0)
)

Schritt 6: Das vollständige Skript¶

Jetzt fügen wir alle Bausteine zu einem lauffähigen Skript zusammen. Alles zusammen in eine Datei, zum Beispiel kugelbahn.py:

from ursina import *
import math

# --- Anwendung ---
app = Ursina(title='Kugelbahn: Einfache Rampe')
window.size = Vec2(1200, 700)
window.color = color.white

# --- Parameter ---
NEIGUNGSWINKEL = 15.0
BAHNLAENGE     = 1.2
KUGELRADIUS    = 0.04
G              = 9.81
A_X            = G * math.sin(math.radians(NEIGUNGSWINKEL))
S_EFF          = BAHNLAENGE - 2 * KUGELRADIUS
START_X        = -(BAHNLAENGE / 2) + KUGELRADIUS
ENDE_X         =  (BAHNLAENGE / 2) - KUGELRADIUS

# --- Objekte ---
Entity(model='cube', color=color.gray,
       scale=Vec3(BAHNLAENGE, 0.02, 0.12), position=Vec3(0, 0, 0))

kugel = Entity(model='sphere', color=color.red,
               scale=2*KUGELRADIUS,
               position=Vec3(START_X, KUGELRADIUS + 0.01, 0))

anzeige = Text(text='', position=(-0.85, 0.45), scale=1.2, color=color.black)

EditorCamera()

# --- Anfangszustand ---
geschwindigkeit = 0.0
t_sim           = 0.0
laeuft          = True

# --- Animationsschleife ---
def update():
    global geschwindigkeit, t_sim, laeuft

    if not laeuft:
        return
    
    # Abbruch
    if kugel.x >= ENDE_X:
        laeuft = False
        print(f'Bewegungszeit: {t_sim:.4f} s')
        print(f'Endgeschwindigkeit: {geschwindigkeit:.4f} m/s')

    # Euler-Cromer-Verfahren: erst Geschwindigkeit, dann Position
    geschwindigkeit += A_X * time.dt
    kugel.x         += geschwindigkeit * time.dt
    t_sim           += time.dt

    # Anzeige
    anzeige.text = f't = {t_sim:.3f} s | v = {geschwindigkeit:.3f} m/s'

app.run()

Die Reihenfolge “erst Geschwindigkeit, dann Position” ist das Euler-Cromer-Verfahren (symplektischer Euler). Es ist energieerhaltender als der klassische explizite Euler, bei dem die Position mit der alten Geschwindigkeit aktualisiert würde. Für die gerade Rampe ist der Unterschied gering, aber die Gewohnheit, in dieser Reihenfolge zu schreiben, zahlt sich in Kapitel 9 aus.

Checkliste: Was prüfen wir nach dem ersten Lauf?

  • Richtung stimmt: Die Kugel bewegt sich nach rechts.

  • Startposition stimmt: Die Kugel beginnt am linken Ende der Rampe.

  • Beschleunigung sichtbar: Die Kugel wird im Verlauf schneller.

  • Anzeige läuft: Zeit und Geschwindigkeit werden aktualisiert.

  • Konsole: Nach dem Stopp erscheinen Bewegungszeit und Endgeschwindigkeit.

Schritt 7: Ergebnis analytisch prüfen¶

NEIGUNGSWINKEL = 15.0
G              = 9.81
BAHNLAENGE     = 1.2
KUGELRADIUS    = 0.04

S_EFF = BAHNLAENGE - 2 * KUGELRADIUS   # 1.12 m

A_X       = G * math.sin(math.radians(NEIGUNGSWINKEL))
V_END     = math.sqrt(2 * A_X * S_EFF)
T_END     = V_END / A_X

print(f"Effektive Wegstrecke S_EFF:     {S_EFF:.3f} m")
print(f"Hangabtriebsbeschleunigung:     {A_X:.3f} m/s²")
print(f"Analytische Endgeschwindigkeit: {V_END:.3f} m/s")
print(f"Analytische Bewegungszeit:      {T_END:.3f} s")
Effektive Wegstrecke S_EFF:     1.120 m
Hangabtriebsbeschleunigung:     2.539 m/s²
Analytische Endgeschwindigkeit: 2.385 m/s
Analytische Bewegungszeit:      0.939 s

In der analytischen Rechnung verwenden wir S_EFF = BAHNLAENGE - 2 · KUGELRADIUS, weil der Kugelmittelpunkt von START_X bis ENDE_X läuft und nicht über die volle Quader-Länge. Eine Restabweichung zwischen Simulation und analytischem Wert entsteht durch den Diskretisierungsfehler der Euler-Cromer-Methode sowie das leichte Überschießen beim Schleifen-Abbruch.

Mini-Übung

Verändern Sie NEIGUNGSWINKEL auf 30° und dann auf 5°. Beobachten Sie die Simulation und berechnen Sie für beide Winkel die analytischen Werte. Entsprechen die Änderungen Ihrer Intuition?

Lösung
import math

G           = 9.81
BAHNLAENGE  = 1.2
KUGELRADIUS = 0.04
S_EFF       = BAHNLAENGE - 2 * KUGELRADIUS

for winkel in [5, 15, 30]:
    a        = G * math.sin(math.radians(winkel))
    v_end    = math.sqrt(2 * a * S_EFF)
    t_bew    = v_end / a
    print(f"θ = {winkel:2d}°: a = {a:.2f} m/s², "
          f"v_end = {v_end:.2f} m/s, t_bew = {t_bew:.2f} s")

Ausgabe:

θ =  5°: a = 0.85 m/s², v_end = 1.38 m/s, t_bew = 1.62 s
θ = 15°: a = 2.54 m/s², v_end = 2.38 m/s, t_bew = 0.94 s
θ = 30°: a = 4.90 m/s², v_end = 3.31 m/s, t_bew = 0.68 s

Eine Verdoppelung des Winkels verdoppelt die Beschleunigung nicht, da sin(θ) nicht linear ist. Die Endgeschwindigkeit hängt von der Wurzel der Beschleunigung ab, die Bewegungszeit nimmt stärker ab als die Endgeschwindigkeit zunimmt.

Zusammenfassung und Ausblick¶

In diesem Abschnitt haben wir eine vollständige Ursina-Simulation von Grund auf aufgebaut: Szene einrichten, Parameter definieren, Objekte platzieren, Anzeige einblenden und den Animations-Loop mit dem Euler-Cromer-Verfahren schreiben. Die effektive Wegstrecke S_EFF stellt sicher, dass der analytische Vergleich geometrisch konsistent ist. Eine Restabweichung durch den Diskretisierungsfehler ist normal.

Im nächsten Abschnitt folgen die Übungen zu Kapitel 8. Kapitel 9 führt dann die vollständige Physik des Rollens ein: Normalkraft, Haft- und Gleitreibung, und die Analogie zwischen translatorischer und rotatorischer Bewegung.
