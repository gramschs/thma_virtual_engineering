````{admonition} Übung 8.2 (✩)
:class: tip
**Animations-Loop analysieren**

Betrachten Sie den folgenden Code:

```python
from vpython import *

kugel = sphere(pos=vec(-0.5, 0.05, 0), radius=0.05, color=color.blue)

v  = vec(0.4, 0, 0)
dt = 0.02
t  = 0.0

while t < 4.0:
    rate(50)
    kugel.pos += v * dt
    t         += dt
```

Beantworten Sie ohne den Code auszuführen:

1. Wie viele Simulationsschritte werden insgesamt ausgeführt?
2. Welche Gesamtstrecke legt die Kugel zurück?
3. An welcher x-Position befindet sich die Kugel am Ende der Simulation?
4. Warum ist `rate(50)` bei `dt = 0.02 s` eine sinnvolle Kombination?
````

````{admonition} Lösung
:class: tip
:class: dropdown
1. **Anzahl Schritte:** `t` läuft von 0 bis 4.0 in Schritten von 0.02 s →
   4.0 / 0.02 = **200 Schritte**.

2. **Gesamtstrecke:** `s = v · t = 0.4 m/s · 4.0 s = **1.6 m** in x-Richtung.

3. **Endposition:** Start bei x = −0.5 m, Verschiebung +1.6 m →
   Endposition x = **+1.1 m**.

4. **rate(50) und dt = 0.02 s:** 50 Schritte pro Sekunde × 0.02 s pro Schritt =
   1.0 simulierte Sekunde pro echter Sekunde. Die Simulation läuft damit in
   Echtzeit.
````

````{admonition} Übung 8.3 (✩✩)
:class: tip
**Freier Fall visualisieren**

Implementieren Sie in VPython eine Simulation des freien Falls. Eine Kugel
startet aus der Ruhe bei `pos = vec(0, 1.0, 0)` und fällt unter dem Einfluss
der Schwerkraft nach unten. Die Simulation soll enden, wenn die Kugel den Boden
erreicht (y = 0 + Kugelradius).

Anforderungen:

1. Erstellen Sie einen Boden (flacher grauer Quader).
2. Starten Sie die Kugel aus dem Stillstand bei y = 1.0 m.
3. Verwenden Sie `g = 9.81 m/s²` und `dt = 0.005 s`.
4. Aktivieren Sie `make_trail`.
5. Geben Sie nach dem Ende der Simulation die Fallzeit und die Auftreffgeschwindigkeit aus.
6. Vergleichen Sie diese Werte mit den analytischen Formeln:
   `t_fall = sqrt(2·h/g)` und `v_auf = g · t_fall`.
````

````{admonition} Lösung
:class: tip
:class: dropdown
```python
from vpython import *

scene.background = color.white
scene.range      = 0.8

KUGELRADIUS = 0.04
STARTHOEHE  = 1.0
g           = 9.81

boden = box(pos=vec(0, -0.01, 0), size=vec(1.5, 0.02, 0.5), color=color.gray(0.6))
kugel = sphere(
    pos        = vec(0, STARTHOEHE, 0),
    radius     = KUGELRADIUS,
    color      = color.orange,
    make_trail = True
)

geschwindigkeit = vec(0, 0, 0)
dt = 0.005
t  = 0.0

while kugel.pos.y > KUGELRADIUS:
    rate(200)
    geschwindigkeit += vec(0, -g, 0) * dt
    kugel.pos       += geschwindigkeit * dt
    t               += dt

v_aufprall = mag(geschwindigkeit)
print(f"Simulierte Fallzeit:           {t:.3f} s")
print(f"Simulierte Auftreffgeschwindigkeit: {v_aufprall:.3f} m/s")

import math
t_analytisch = math.sqrt(2 * STARTHOEHE / G)
v_analytisch = g * t_analytisch
print(f"Analytische Fallzeit:           {t_analytisch:.3f} s")
print(f"Analytische Auftreffgeschwindigkeit: {v_analytisch:.3f} m/s")
```

Die Simulation stimmt sehr gut mit der analytischen Lösung überein. Eine kleine
Abweichung durch die Euler-Methode ist bei `dt = 0.005 s` kaum messbar.
````

````{admonition} Übung 8.5 (✩✩✩)
:class: tip
**Parametervergleich: Neigungswinkel und Rollzeit**

Simulieren Sie die Kugel aus Abschnitt 8.3 für drei verschiedene
Neigungswinkel: 5°, 15° und 30°. Führen Sie alle drei Simulationen
nacheinander mit unterschiedlichen Kugelfarben durch und stellen Sie die
Ergebnisse anschließend in einem Plotly-Diagramm dar.

Anforderungen:

1. Verwenden Sie das vollständige Modell aus Abschnitt 8.3 (Euler-Methode,
   `dt = 0.005 s`, Bahnlänge 1.2 m).
2. Speichern Sie für jede Simulation in einer Liste: Zeitpunkte `t_liste` und
   zugehörige x-Positionen `x_liste`.
3. Zeichnen Sie alle drei Kurven in einem einzigen Plotly-Diagramm
   (x-Achse: Zeit in Sekunden, y-Achse: Position in Metern).
4. Beschriften Sie jede Kurve mit dem Neigungswinkel.
5. Lesen Sie aus dem Diagramm ab: Bei welchem Winkel ist die Rollzeit am
   kürzesten, und um welchen Faktor unterscheidet sie sich von der langsamsten?
````

````{admonition} Lösung
:class: tip
:class: dropdown
```python
import math
import plotly.express as px
import pandas as pd

g          = 9.81
BAHNLAENGE = 1.2
KUGELRADIUS = 0.04
dt         = 0.005
START_X    = -(BAHNLAENGE / 2) + KUGELRADIUS
ENDE_X     = (BAHNLAENGE / 2) - KUGELRADIUS

WINKEL   = [5, 15, 30]
ergebnis = []

for winkel in WINKEL:
    a_x             = g * math.sin(math.radians(winkel))
    geschwindigkeit = 0.0
    x               = START_X
    t               = 0.0

    while x < ENDE_X:
        geschwindigkeit += a_x * dt
        x               += geschwindigkeit * dt
        t               += dt
        ergebnis.append({"Winkel": f"{winkel}°", "Zeit (s)": round(t, 4), "Position (m)": round(x, 5)})

df = pd.DataFrame(ergebnis)

fig = px.line(
    df,
    x     = "Zeit (s)",
    y     = "Position (m)",
    color = "Winkel",
    title = "Kugelposition auf der Rampe: Vergleich Neigungswinkel",
    labels = {"Zeit (s)": "Zeit t (s)", "Position (m)": "Position x (m)"}
)
fig.show()

# Rollzeiten ausgeben
for winkel in WINKEL:
    subset = df[df["Winkel"] == f"{winkel}°"]
    t_end  = subset["Zeit (s)"].max()
    print(f"θ = {winkel:2d}°: Rollzeit = {t_end:.3f} s")
```

Bei 30° ist die Rollzeit am kürzesten, bei 5° am längsten. Das Verhältnis
beträgt etwa Faktor 2.4: Die flache Rampe braucht mehr als doppelt so lange.
Im Diagramm ist gut erkennbar, dass die steilere Kurve schneller die Position
x = 0.56 m (Ende der Bahn) erreicht und dabei eine deutlich stärkere Krümmung
aufweist.
````
