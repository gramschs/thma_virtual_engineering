# 9.2 Wie implementiere ich ein physikalisches Modell in Python?

Wir kennen jetzt die Kräfte: Hangabtrieb, Normalkraft, Haftreibung,
Gleitreibung. Und wir wissen, wann das Objekt stehen bleibt und wann es sich

## Vom Kraftdiagramm zum Code

In Abschnitt 9.1 haben wir das Kraftdiagramm aufgestellt. Jetzt übersetzen wir
es Schritt für Schritt in Python. Wir beginnen mit dem translatorischen Pfad.

### Schritt 1: Parameter definieren

```{code-cell} python
import math

# Objektparameter
m    = 0.1     # Masse in kg
G    = 9.81    # Erdbeschleunigung in m/s²
THETA = math.radians(20)   # Neigungswinkel in Radiant

# Reibungskoeffizienten
MU_H = 0.35   # Haftreibungskoeffizient (static friction)
MU_G = 0.25   # Gleitreibungskoeffizient (kinetic friction)

# Kräfte
F_N        = m * G * math.cos(THETA)   # Normalkraft
F_H        = m * G * math.sin(THETA)   # Hangabtriebskraft
F_HAFT_MAX = MU_H * F_N                # Maximale Haftreibungskraft

print(f"F_N        = {F_N:.4f} N")
print(f"F_H        = {F_H:.4f} N")
print(f"F_Haft,max = {F_HAFT_MAX:.4f} N")
print(f"Bewegt sich das Objekt? {'Ja' if F_H > F_HAFT_MAX else 'Nein'}")
```

### Schritt 2: Den Zustandsübergang modellieren

Das Herzstück des Modells ist der Zustandsübergang zwischen Haftreibung und
Gleitreibung. Wir modellieren ihn mit einer booleschen Variable `gleitet`, die
angibt, in welchem Zustand sich das Objekt gerade befindet:

```{code-cell} python
import math

m    = 0.1
G    = 9.81
THETA = math.radians(20)
MU_H = 0.35
MU_G = 0.25

F_N        = m * G * math.cos(THETA)
F_H        = m * G * math.sin(THETA)
F_HAFT_MAX = MU_H * F_N

# Anfangszustand: Objekt steht still
gleitet     = False
geschwindigkeit = 0.0

# Zustandsübergang prüfen
if not gleitet and F_H > F_HAFT_MAX:
    gleitet = True
    print("Haftreibung überwunden – Objekt beginnt zu gleiten.")

# Beschleunigung berechnen
if gleitet:
    F_GLEIT = MU_G * F_N
    F_netto = F_H - F_GLEIT
    a       = F_netto / m
else:
    a = 0.0   # Objekt steht still

print(f"Beschleunigung: {a:.4f} m/s²")
```

*Warum setzen wir `gleitet` nicht einfach direkt auf `True` oder `False`?*

Weil der Zustand während der Simulation wechseln kann. Das Objekt kann stehen,
dann zu gleiten beginnen und – wenn eine äußere Kraft nachlässt oder die Bahn
flacher wird – theoretisch wieder zum Stillstand kommen. Die Variable `gleitet`
verfolgt diesen Zustand über alle Zeitschritte hinweg.

### Schritt 3: Den vollständigen Euler-Loop aufbauen

Jetzt fügen wir den Zustandsübergang in den Loop aus Kapitel 8 ein:

```{code-cell} python
import math
import pandas as pd
import plotly.express as px

# Parameter
m     = 0.1
G     = 9.81
THETA = math.radians(20)
MU_H  = 0.35
MU_G  = 0.25
BAHNLAENGE = 1.2       # Länge der Rampe in m
dt         = 0.005     # Zeitschritt in s

# Kräfte (zeitunabhängig bei konstanter Neigung)
F_N        = m * G * math.cos(THETA)
F_H        = m * G * math.sin(THETA)
F_HAFT_MAX = MU_H * F_N
F_GLEIT    = MU_G * F_N

# Anfangszustand
x               = 0.0
geschwindigkeit = 0.0
gleitet         = False
t               = 0.0

# Ergebnislisten
ergebnis = []

while x < BAHNLAENGE:
    # Zustandsübergang: Haftreibung → Gleitreibung
    if not gleitet and F_H > F_HAFT_MAX:
        gleitet = True

    # Beschleunigung
    if gleitet:
        a = (F_H - F_GLEIT) / m
    else:
        a = 0.0

    # Euler-Schritt
    geschwindigkeit += a * dt
    x               += geschwindigkeit * dt
    t               += dt

    ergebnis.append({
        "Zeit (s)":        round(t, 4),
        "Position (m)":    round(x, 5),
        "Geschwindigkeit (m/s)": round(geschwindigkeit, 5),
        "Zustand":         "Gleiten" if gleitet else "Haften"
    })

df = pd.DataFrame(ergebnis)

# Diagramm: Position über Zeit
fig = px.line(
    df, x="Zeit (s)", y="Position (m)",
    title=f"Position der Kugel (θ={math.degrees(THETA):.0f}°, μ_H={MU_H}, μ_G={MU_G})",
    color="Zustand"
)
fig.show()

print(f"\nRollzeit:          {t:.3f} s")
print(f"Endgeschwindigkeit: {geschwindigkeit:.3f} m/s")
```

```{admonition} Mini-Übung
:class: tip
Setzen Sie `MU_H = 0.15`. Was beobachten Sie beim Zustandsübergang? Erklären
Sie, warum die Kurve nun anders aussieht als bei `MU_H = 0.35`.
```

````{admonition} Lösung
:class: tip
:class: dropdown
Bei `MU_H = 0.15` ist die maximale Haftreibungskraft kleiner als die
Hangabtriebskraft – das Objekt gleitet sofort von Anfang an. Der Zustand
"Haften" tritt gar nicht erst auf. Die Kurve beginnt daher sofort mit
konstanter Beschleunigung ohne einen flachen Startabschnitt.

Bei `MU_H = 0.35` und θ = 20° ist `F_Haft,max = 0.35 · m · g · cos(20°) >
F_H = m · g · sin(20°)`, sodass das Objekt stehen bleibt. Es bewegt sich nur,
wenn der Winkel erhöht oder der Koeffizient verringert wird.
````

## Das Modell analytisch überprüfen

Für eine gleichmäßig beschleunigte Bewegung ohne Haftreibung (das Objekt
gleitet sofort) gibt es eine exakte Lösung:

```
a    = g · (sin(θ) - μ_G · cos(θ))
t    = sqrt(2 · s / a)
v_end = a · t
```

Wir vergleichen diese analytische Lösung mit unserem Simulationsergebnis:

```{code-cell} python
import math

G     = 9.81
THETA = math.radians(20)
MU_G  = 0.25
s     = 1.2       # Bahnlänge

a_analytisch    = G * (math.sin(THETA) - MU_G * math.cos(THETA))
t_analytisch    = math.sqrt(2 * s / a_analytisch)
v_end_analytisch = a_analytisch * t_analytisch

print(f"Analytische Beschleunigung:    {a_analytisch:.4f} m/s²")
print(f"Analytische Rollzeit:          {t_analytisch:.4f} s")
print(f"Analytische Endgeschwindigkeit: {v_end_analytisch:.4f} m/s")
```

Vergleichen wir diese Werte mit den Simulationsergebnissen aus dem Loop oben.
Bei `dt = 0.005 s` sollte die Abweichung unter 1 % liegen.

## Der rotatorische Pfad: Drehbewegung unter Drehmoment

Für Objekte, die sich um eine feste Achse drehen – Fahrradkurbel, Zahnrad –
ist die Struktur identisch. Die Zustandsgrößen wechseln, das Grundgerüst bleibt.

### Parameter und Kräfte

Statt Masse und Kraft arbeiten wir mit Trägheitsmoment und Drehmoment:

```{code-cell} python
import math

# Rotatorische Parameter
I         = 0.002    # Trägheitsmoment in kg·m² (als gegebener Parameter)
M_ANTRIEB = 0.05     # Antriebsdrehmoment in N·m (zum Beispiel durch eine Feder)
MU_ROT    = 0.08     # Reibungskoeffizient (dimensionslos, vereinfacht)
r         = 0.15     # Effektiver Hebelarm in m (Kurbelradius)

# Reibungsdrehmoment (analog zu F_GLEIT = μ_G · F_N)
# Vereinfacht: M_Reib = μ_ROT · M_ANTRIEB
M_REIB = MU_ROT * M_ANTRIEB

print(f"Antriebsdrehmoment:  {M_ANTRIEB:.4f} N·m")
print(f"Reibungsdrehmoment:  {M_REIB:.4f} N·m")
print(f"Nettodrehmoment:     {M_ANTRIEB - M_REIB:.4f} N·m")
```

```{admonition} Hinweis: Trägheitsmoment als Parameter
:class: note
Das Trägheitsmoment I hängt von der Massenverteilung des Objekts ab und ist
für komplexe Geometrien aufwändig zu berechnen. Für Kapitel 9 entnehmen wir I
einer Tabelle oder messen es experimentell. Typische Werte für einfache Körper:

| Körper | Trägheitsmoment |
| ------ | --------------- |
| Vollzylinder (Radius r, Masse m) | I = ½ · m · r² |
| Hohlzylinder (Innen r₁, Außen r₂) | I = ½ · m · (r₁² + r₂²) |
| Dünne Scheibe (Radius r, Masse m) | I = ½ · m · r² |
| Dünner Stab (Länge L, Masse m, um Mittelpunkt) | I = 1/12 · m · L² |
```

### Der rotatorische Euler-Loop

```{code-cell} python
import math
import pandas as pd
import plotly.express as px

# Parameter
I         = 0.002
M_ANTRIEB = 0.05
MU_ROT    = 0.08
M_REIB    = MU_ROT * M_ANTRIEB
WINKEL_ZIEL = 4 * math.pi   # zwei volle Umdrehungen
dt        = 0.005

# Anfangszustand
omega   = 0.0    # Winkelgeschwindigkeit in rad/s
theta   = 0.0    # Winkel in Radiant
t       = 0.0

# Haftreibung: Bewegt sich die Kurbel überhaupt?
M_HAFT_MAX = 1.2 * M_REIB   # Haftreibungsmoment etwas größer als Gleitreibungsmoment
dreht      = False

ergebnis = []

while theta < WINKEL_ZIEL:
    # Zustandsübergang
    if not dreht and M_ANTRIEB > M_HAFT_MAX:
        dreht = True

    # Winkelbeschleunigung
    if dreht:
        M_netto = M_ANTRIEB - M_REIB
        alpha   = M_netto / I
    else:
        alpha = 0.0

    # Euler-Schritt (rotatorisch)
    omega += alpha * dt
    theta += omega * dt
    t     += dt

    ergebnis.append({
        "Zeit (s)":                   round(t, 4),
        "Winkel (rad)":               round(theta, 5),
        "Winkelgeschwindigkeit (rad/s)": round(omega, 5),
        "Zustand":                    "Dreht" if dreht else "Haftend"
    })

df = pd.DataFrame(ergebnis)

fig = px.line(
    df, x="Zeit (s)", y="Winkelgeschwindigkeit (rad/s)",
    title="Winkelgeschwindigkeit der Kurbel über die Zeit",
    color="Zustand"
)
fig.show()

print(f"\nZeit für zwei Umdrehungen: {t:.3f} s")
print(f"Endwinkelgeschwindigkeit:  {omega:.3f} rad/s")
print(f"Enddrehzahl:               {omega / (2*math.pi) * 60:.1f} U/min")
```

```{admonition} Mini-Übung
:class: tip
Vergleichen Sie die Struktur des rotatorischen Loops mit dem translatorischen

Variablen und Parametern der beiden Pfade.
```

````{admonition} Lösung
:class: tip
:class: dropdown
| Translatorisch | Rotatorisch |
| -------------- | ----------- |
| `m` (Masse) | `I` (Trägheitsmoment) |
| `F_H - F_GLEIT` (Nettokraft) | `M_ANTRIEB - M_REIB` (Nettodrehmoment) |
| `geschwindigkeit` | `omega` (Winkelgeschwindigkeit) |
| `x` (Position) | `theta` (Winkel) |

Die Euler-Schritte sind strukturell identisch:
```python
# Translation:       # Rotation:
a = F / m            alpha = M / I
v += a * dt          omega += alpha * dt
x += v * dt          theta += omega * dt
```
````

## Beide Pfade im Vergleich

Um den Überblick zu behalten, stellen wir die beiden Implementierungen
nebeneinander. Die einzigen Unterschiede sind die Variablennamen und die
Bedeutung der Parameter – die Struktur ist vollständig identisch:

```{code-cell} python
# Translatorischer Pfad – Kurzform
def simulate_translation(m, F_netto, x_end, dt=0.005):
    v, x, t = 0.0, 0.0, 0.0
    while x < x_end:
        a  = F_netto / m
        v += a * dt
        x += v * dt
        t += dt
    return t, v

# Rotatorischer Pfad – Kurzform
def simulate_rotation(I, M_netto, theta_end, dt=0.005):
    omega, theta, t = 0.0, 0.0, 0.0
    while theta < theta_end:
        alpha  = M_netto / I
        omega += alpha * dt
        theta += omega * dt
        t     += dt
    return t, omega

import math

# Translatorisches Beispiel: Kugelbahn, θ=20°, μ_G=0.25
G     = 9.81
THETA = math.radians(20)
m     = 0.1
MU_G  = 0.25
F_net_trans = m * G * (math.sin(THETA) - MU_G * math.cos(THETA))
t_trans, v_end = simulate_translation(m, F_net_trans, x_end=1.2)
print(f"Translation – Rollzeit: {t_trans:.3f} s, Endgeschwindigkeit: {v_end:.3f} m/s")

# Rotatorisches Beispiel: Kurbel
I         = 0.002
M_netto   = 0.05 - 0.08 * 0.05
theta_end = 4 * math.pi
t_rot, omega_end = simulate_rotation(I, M_netto, theta_end)
print(f"Rotation   – Zeit:      {t_rot:.3f} s, Endwinkelgeschwindigkeit: {omega_end:.3f} rad/s")
```

## Zusammenfassung und Ausblick

In diesem Abschnitt haben wir das Kraftmodell aus Abschnitt 9.1 in Python
implementiert. Das zentrale Werkzeug ist der Zustandsübergang von Haftreibung
zu Gleitreibung als `if`/`else`-Konstrukt: Solange die Hangabtriebskraft die
maximale Haftreibungskraft nicht übersteigt, ist die Beschleunigung null. Sobald
sie es tut, wechseln wir in den Gleitreibungszustand und berechnen die
Nettokraft. Der rotatorische Pfad für Objekte wie Kurbeln und Zahnräder folgt
derselben Struktur, nur mit Drehmoment statt Kraft und Trägheitsmoment statt
Masse.

Im nächsten Abschnitt verbinden wir dieses Physikmodell mit VPython: Die Kugel
bewegt sich mit korrekter Reibungsphysik, Kräftepfeile aktualisieren sich in
Echtzeit, und eine Anzeige zeigt den aktuellen Zustand (Haften oder Gleiten).
