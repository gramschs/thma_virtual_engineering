# 10.1 Wie beschreibe ich eine komplexe Bahn geometrisch?

Die Simulation aus Kapitel 9 läuft. Die Kugel beschleunigt, die Reibung bremst
sie, und am Ende der Rampe stimmt die Geschwindigkeit gut mit dem analytischen
Wert überein. Aber wenn wir unsere echte Kugelbahn anschauen, fällt sofort auf:
Sie ist keine gerade Rampe. Sie hat Kurven, wechselnde Steigungen, vielleicht
einen Bogen oder eine Spirale. Eine einzige Zahl für den Neigungswinkel reicht
nicht aus, um diese Geometrie zu beschreiben. Wir brauchen ein allgemeineres
Konzept.

## Was ist eine gerade Rampe nicht?

Stellen wir uns vor, wir fahren mit dem Auto einen Bergpass hinauf. Die Straße
hat keine konstante Steigung – sie flacht an manchen Stellen ab, wird an
anderen steiler, und zwischendurch gibt es Kurven. Niemand würde die Strecke
mit einem einzigen Neigungswinkel beschreiben. Stattdessen teilen wir sie in
Abschnitte auf: Hier ist die Steigung 8 %, dort 4 %, und in diesem Abschnitt
geht es kurz bergab. Genau dieses Prinzip wenden wir auf unsere Kugelbahn an.

*Aber wozu brauchen wir die genaue Geometrie überhaupt? Reicht nicht eine
mittlere Steigung?*

Eine mittlere Steigung würde die Gesamtrollzeit vielleicht annähernd richtig
vorhersagen. Aber sie würde nicht erklären, wo die Kugel besonders schnell
wird, wo sie abbremst und ob sie an einer flachen Stelle kurz zum Stillstand
kommen könnte. Für eine Validierung mit Phyphox-Messdaten (Kapitel 10.2)
brauchen wir die zeitaufgelöste Geschwindigkeit – und dafür müssen wir die
Geometrie korrekt modellieren.

## Wegpunkte: Die Bahn als Punktefolge

Die einfachste Möglichkeit, eine komplexe Bahn zu beschreiben, ist eine
geordnete Liste von Punkten im Raum. Wir nennen sie **Wegpunkte** (englisch:
*waypoints*). Zwischen je zwei aufeinanderfolgenden Wegpunkten nehmen wir an,
dass die Bahn geradlinig verläuft. Das Ergebnis ist eine stückweise lineare
Approximation der echten Bahn.

Ein Wegpunkt ist ein Tripel `(x, y, z)` in Metern. Für eine ebene Bahn, die
sich nur in der x-z-Ebene erstreckt, reicht `(x, y)`. Für unsere räumliche
Kugelbahn brauchen wir alle drei Koordinaten.

```{code-cell} python
import numpy as np

# Beispielhafte Wegpunkte einer vereinfachten Kugelbahn
# Format: (x, y, z) in Metern, von Start bis Ende
wegpunkte = np.array([
    [0.00,  0.00, 0.00],   # Start
    [0.20, -0.03, 0.00],   # erstes Gefälle
    [0.40, -0.05, 0.02],   # leichte Kurve nach rechts
    [0.60, -0.06, 0.04],   # flaches Mittelstück
    [0.80, -0.09, 0.03],   # zweites Gefälle
    [1.00, -0.12, 0.01],   # Auslauf
])

print(f"Anzahl Wegpunkte:  {len(wegpunkte)}")
print(f"Anzahl Segmente:   {len(wegpunkte) - 1}")
```

### Segmentlänge: Wie lang ist ein Abschnitt?

Die Länge eines Segments zwischen Wegpunkt `i` und Wegpunkt `i+1` berechnet
sich nach dem Satz des Pythagoras im dreidimensionalen Raum:

```
L = sqrt(Δx² + Δy² + Δz²)
```

In Python verwenden wir `np.linalg.norm`, das den Betrag eines
Differenzvektors berechnet:

```{code-cell} python
import numpy as np

wegpunkte = np.array([
    [0.00,  0.00, 0.00],
    [0.20, -0.03, 0.00],
    [0.40, -0.05, 0.02],
    [0.60, -0.06, 0.04],
    [0.80, -0.09, 0.03],
    [1.00, -0.12, 0.01],
])

laengen = []
for i in range(len(wegpunkte) - 1):
    delta  = wegpunkte[i+1] - wegpunkte[i]
    laenge = np.linalg.norm(delta)
    laengen.append(laenge)

laengen         = np.array(laengen)
gesamtlaenge    = np.sum(laengen)

print("Segmentlängen (m):")
for i, l in enumerate(laengen):
    print(f"  Segment {i+1}: {l:.4f} m")
print(f"\nGesamtlänge: {gesamtlaenge:.4f} m")
```

### Lokaler Neigungswinkel: Wie steil ist ein Abschnitt?

Der lokale Neigungswinkel eines Segments beschreibt, wie stark die Bahn in
diesem Abschnitt ansteigt oder abfällt. Er ergibt sich aus dem Höhenunterschied
`Δy` und der horizontalen Distanz `Δd = sqrt(Δx² + Δz²)`:

```
θ = arctan(Δy / Δd)
```

Ein negativer Winkel bedeutet ein Gefälle (die Kugel fährt bergab), ein
positiver Winkel ein Anstieg.

```{code-cell} python
import numpy as np

wegpunkte = np.array([
    [0.00,  0.00, 0.00],
    [0.20, -0.03, 0.00],
    [0.40, -0.05, 0.02],
    [0.60, -0.06, 0.04],
    [0.80, -0.09, 0.03],
    [1.00, -0.12, 0.01],
])

winkel_grad = []
for i in range(len(wegpunkte) - 1):
    delta      = wegpunkte[i+1] - wegpunkte[i]
    delta_horiz = np.sqrt(delta[0]**2 + delta[2]**2)
    theta_rad  = np.arctan2(delta[1], delta_horiz)   # Vorzeichen korrekt
    winkel_grad.append(np.degrees(theta_rad))

print("Lokale Neigungswinkel (°):")
for i, w in enumerate(winkel_grad):
    richtung = "Gefälle" if w < 0 else "Anstieg" if w > 0 else "horizontal"
    print(f"  Segment {i+1}: {w:+.2f}° ({richtung})")
```

## Die Bahn visualisieren

Bevor wir die Wegpunkte in die Simulation übergeben, visualisieren wir sie mit
Plotly, um sie auf Plausibilität zu prüfen. Eine unerwartete Steigung oder ein
falsch gesetzter Wegpunkt fällt hier sofort auf.

```{code-cell} python
import numpy as np
import plotly.graph_objects as go

wegpunkte = np.array([
    [0.00,  0.00, 0.00],
    [0.20, -0.03, 0.00],
    [0.40, -0.05, 0.02],
    [0.60, -0.06, 0.04],
    [0.80, -0.09, 0.03],
    [1.00, -0.12, 0.01],
])

fig = go.Figure()

# Bahn als Linie
fig.add_trace(go.Scatter3d(
    x=wegpunkte[:, 0], y=wegpunkte[:, 2], z=wegpunkte[:, 1],
    mode="lines+markers",
    line=dict(color="royalblue", width=4),
    marker=dict(size=5, color="tomato"),
    name="Bahn"
))

fig.update_layout(
    title="Kugelbahn – Wegpunkte und Segmente",
    scene=dict(
        xaxis_title="x (m)",
        yaxis_title="z (m)",
        zaxis_title="y – Höhe (m)",
        aspectmode="data"
    )
)
fig.show()
```

## Wegpunkte aus CloudCompare extrahieren

Unser Musterdatensatz enthält das bereinigte Mesh der Kugelbahn aus Kapitel 4
bis 6. Statt die Wegpunkte von Hand zu definieren, entnehmen wir sie direkt
aus dem Mesh. CloudCompare bietet dafür zwei praktische Möglichkeiten.

### Manuell: Punkte entlang der Führungsrille aufnehmen

Die zuverlässigste Methode ist das manuelle Aufnehmen von Punkten entlang der
Führungsrille im 3D-Viewer von CloudCompare:

```{admonition} Schritt-für-Schritt: Wegpunkte in CloudCompare aufnehmen
:class: note
1. Das bereinigte Mesh der Kugelbahn in CloudCompare laden.
2. Im Menü `Tools > Point picking` aktivieren (`Shift + P`).
3. Mit der Maus nacheinander Punkte entlang der Mittellinie der Führungsrille
   anklicken – etwa alle 5 cm einen Punkt.
4. Nach dem letzten Punkt auf "Export to ASCII" klicken.
5. Als Trennzeichen Komma wählen, Spaltenreihenfolge: x, y, z.
6. Datei als `wegpunkte_kugelbahn.csv` speichern.
```

## Die Wegpunktliste als Schnittstelle zur Simulation

Am Ende dieses Abschnitts haben wir eine Datenstruktur, die wir direkt in die
Simulation von Abschnitt 10.3 übergeben können:

```{code-cell} python
import numpy as np

def bahn_aus_wegpunkten(wegpunkte):
    """
    Berechnet Segmentlängen und lokale Neigungswinkel aus einer Wegpunktliste.

    Parameter:
        wegpunkte : np.ndarray, Shape (N, 3) – Wegpunkte als (x, y, z) in Metern

    Rückgabe:
        laengen     : np.ndarray, Shape (N-1,) – Segmentlängen in Metern
        winkel_rad  : np.ndarray, Shape (N-1,) – Neigungswinkel in Radiant
    """
    laengen    = []
    winkel_rad = []

    for i in range(len(wegpunkte) - 1):
        delta       = wegpunkte[i+1] - wegpunkte[i]
        laenge      = np.linalg.norm(delta)
        delta_horiz = np.sqrt(delta[0]**2 + delta[2]**2)
        theta       = np.arctan2(delta[1], delta_horiz)
        laengen.append(laenge)
        winkel_rad.append(theta)

    return np.array(laengen), np.array(winkel_rad)


# Verwendung:
wegpunkte          = np.array([
    [0.00, 0.00, 0.00], [0.20, -0.03, 0.00],
    [0.40, -0.05, 0.02], [0.60, -0.06, 0.04],
    [0.80, -0.09, 0.03], [1.00, -0.12, 0.01],
])
laengen, winkel    = bahn_aus_wegpunkten(wegpunkte)

print("Bahn bereit für Simulation:")
print(f"  {len(laengen)} Segmente, Gesamtlänge {np.sum(laengen):.3f} m")
print(f"  Winkelbereich: {np.degrees(winkel.min()):.1f}° bis {np.degrees(winkel.max()):.1f}°")
```

Diese Funktion `bahn_aus_wegpunkten` werden wir in Abschnitt 10.3 direkt
importieren und in den segmentweisen Euler-Loop einsetzen.

## Zusammenfassung und Ausblick

In diesem Abschnitt haben wir das Konzept der stückweise linearen Bahnbeschreibung
eingeführt. Eine Folge von Wegpunkten im dreidimensionalen Raum definiert
Segmente mit individuellen Längen und Neigungswinkeln. Diese Geometrie lässt
sich in CloudCompare manuell entlang der Führungsrille aufnehmen, als CSV
exportieren und mit wenigen Zeilen Python einlesen. Die Funktion
`bahn_aus_wegpunkten` kapselt die gesamte Geometrieberechnung und liefert die
Schnittstelle zur Simulation.

In Abschnitt 10.2 verlassen wir kurz die Simulation und schauen uns an, wie
wir die Bewegung des echten Objekts messen – mit Stoppuhr und Maßband oder mit
der Phyphox-App – und welche Gütemaße wir verwenden, um Simulation und Messung
quantitativ zu vergleichen.
