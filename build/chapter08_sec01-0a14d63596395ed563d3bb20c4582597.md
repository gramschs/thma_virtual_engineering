# 8.1 Was ist Ursina, und wie funktioniert es als 3D-Umgebung?

Wir haben unsere Kugelbahn digitalisiert, bereinigt und gedruckt. Das physische
Objekt liegt vor uns. Doch wie verhält sich eine Kugel, die diese Bahn
hinunterrollt? Wie schnell ist sie an welcher Stelle, und welchen Einfluss hat
die Neigung auf die Rollzeit? Diese Fragen könnten wir mit Stoppuhr und Lineal
beantworten. Wir wählen jedoch einen anderen Weg: Wir bauen ein Computermodell
der Kugelbahn und lassen die Kugel digital rollen.

## Warum reicht Python allein nicht aus?

Stellen wir uns vor, wir berechnen mit Python die Position einer Kugel zum
Zeitpunkt `t = 0.5 s` und erhalten als Ergebnis:

```code
x = 0.32 m,  y = 0.12 m,  z = 0.04 m
```

Drei Zahlen. Korrekt, aber wenig anschaulich. Wir sehen nicht, wo auf der Bahn
sich die Kugel gerade befindet, wie sie sich bewegt oder ob die berechnete Bahn
überhaupt plausibel aussieht. Genau hier setzt **Ursina** an.

Ursina ist eine Python-Bibliothek, die auf der etablierten 3D-Engine Panda3D
aufbaut und interaktive 3D-Szenen als Desktop-Anwendung erzeugt. Wir schreiben
Python-Code, und Ursina öffnet ein Fenster, in dem unsere Simulation sichtbar
wird. Eine Kugel, die in der Simulation ihren Ort ändert, bewegt sich auch im
Bild. Ein markierter Pfeil, der eine Kraft darstellt, zeigt sofort in die
richtige oder in die falsche Richtung.

*Warum ist diese visuelle Rückkopplung so wichtig?*

Ein typischer Fehler in physikalischen Simulationen ist ein falsches Vorzeichen
in der Kraftberechnung. Numerisch ist das Ergebnis eine Zahl; ohne
Visualisierung bemerken wir den Fehler vielleicht nicht. Mit Ursina sehen wir
sofort, wenn die Kugel nach oben statt nach unten beschleunigt. In Kapitel 9
werden wir genau diesen Effekt nutzen, um unsere Kraftberechnungen Schritt für
Schritt zu überprüfen.

## Wie starte ich Ursina?

Ursina wird als Python-Paket über `pip` installiert. Im Gegensatz zu manchen
anderen 3D-Bibliotheken benötigt es keine Browser-Plugins oder
Web-Abhängigkeiten und läuft stabil auf Windows, macOS und Linux.

````{admonition} Installation von Ursina mit Anaconda
:class: warning
Die folgende Installationsanweisung setzt eine Python-Installation mit Anaconda
voraus.


1. Neue Umgebung anlegen: `conda create -n ursina python=3.11`
2. Umgebung aktivieren: `conda activate ursina`

3. Pip aktualisieren: `pip install --upgrade pip`
4. Ursina installieren: `pip install ursina`

Nach der Installation können Sie ein erstes Testprogramm ausführen:

```python
from ursina import *
app = Ursina()
app.run()
```

Es öffnet sich ein leeres 3D-Fenster. Mit `Escape` schließen Sie es.
````

Jedes Ursina-Programm folgt demselben Grundmuster:

```{code-cell} python
:tags: [skip-execution]
from ursina import *

app = Ursina()

### Hier bauen wir die Szene auf

EditorCamera()    # aktiviert Maus-Navigation im 3D-Fenster

app.run()
```

Die Funktion `app.run()` startet die sogenannte **Game Loop** (Spielschleife),
eine Endlosschleife, die das Fenster zeichnet und auf Benutzereingaben reagiert.
Alles, was sich in der Simulation bewegen oder verändern soll, wird innerhalb
dieser Schleife gesteuert. Wie das konkret funktioniert, lernen wir in
Abschnitt 8.2.

## Wie ist das Koordinatensystem aufgebaut?

Ursina verwendet ein linkshändiges kartesisches Koordinatensystem:

- **x** zeigt nach rechts
- **y** zeigt nach oben
- **z** zeigt in den Bildschirm hinein (von der betrachtenden Person weg)

Das unterscheidet sich von der in der Physik üblichen rechtshändigen Konvention,
ist aber in der Computergrafik und in Game-Engines weit verbreitet. Für unsere
Kugelbahn bedeutet das: Die Bahn verläuft hauptsächlich in x-Richtung, die
Schwerkraft wirkt in die negative y-Richtung, und die Breite der Führungsrille
erstreckt sich in z-Richtung.

Alle Positionen und Richtungen in Ursina werden mit Vektoren angegeben. Ein
**Vektor** (englisch: Vector) fasst drei Zahlen zusammen und wird mit `Vec3`
erstellt:

```{code-cell} python
from ursina import Vec3

### Ein Vektor mit x=1, y=0.5, z=0

position = Vec3(1, 0.5, 0)

### Vektoren können addiert, subtrahiert und skaliert werden

versatz       = Vec3(0.2, 0, 0)
neue_position = position + versatz
doppelt       = 2 * position

print(f"Ursprüngliche Position: {position}")
print(f"Neue Position:          {neue_position}")
print(f"Doppelt skaliert:       {doppelt}")
print(f"Betrag (Länge):         {position.length():.3f}")
```

Die Methode `.length()` gibt den Betrag (die Länge) eines Vektors zurück.

## Welche Grundobjekte kennt Ursina?

In Ursina sind alle sichtbaren Objekte sogenannte **Entities** (Objekte oder
Entitäten). Eine Entity erhält beim Erstellen ein Modell, eine Position, eine
Skalierung und eine Farbe. Ursina bringt einige einfache Standard-Modelle mit,
die wir direkt als Zeichenkette angeben können. Genau das ist der Unterschied
zu reinen Berechnungsbibliotheken wie NumPy: Wir beschreiben nicht nur Zahlen,
sondern sichtbare Objekte in einem 3D-Raum.

### Sphere: die Kugel

```{code-cell} python
:tags: [skip-execution]
kugel = Entity(
    model    = 'sphere',
    position = Vec3(0, 0, 0),
    scale    = 0.1,             # Durchmesser = 0.1 Einheiten (Radius 0.05)
    color    = color.red
)
```

`position` ist der Mittelpunkt der Kugel. `scale` gibt den Durchmesser an:
Ein Wert von `0.1` entspricht einem Radius von `0.05`. Das zurückgegebene
Objekt `kugel` können wir später ansprechen, um seine Eigenschaften zu
verändern, zum Beispiel `kugel.position = Vec3(0.1, 0, 0)`.

### Cube: der Quader

```{code-cell} python
:tags: [skip-execution]
rampe = Entity(
    model    = 'cube',
    position = Vec3(0, -0.01, 0),
    scale    = (2.0, 0.02, 0.15),     # Ausdehnung in x, y, z
    color    = color.gray
)
```

`scale` gibt bei einem Quader (englisch: Cube) die Ausdehnungen in x-, y- und
z-Richtung an. Der Quader ist das wichtigste Grundobjekt für Flächen, Wände
und Führungsschienen.

### Cylinder: der Zylinder

```{code-cell} python
:tags: [skip-execution]
stab = Entity(
    model     = 'cylinder',
    position  = Vec3(0, 0, 0),
    scale     = (0.02, 1.0, 0.02),    # Durchmesser x/z, Länge y
    color     = color.blue
)
```

Das Standard-Zylinder-Modell ist entlang der y-Achse ausgerichtet. Um ihn
entlang der x-Achse zu orientieren, fügen wir `rotation_z = 90` hinzu; für
die z-Achse entsprechend `rotation_x = 90`.

## Wie stelle ich die Szene ein?

Das `window`-Objekt steuert Fenstertitel und Hintergrundfarbe. Wir setzen
diese Eigenschaften direkt nach dem Erstellen der Anwendung:

```{code-cell} python
:tags: [skip-execution]
from ursina import *

app = Ursina()

window.title = "Kugelbahn - Simulation"
window.color = color.white

EditorCamera()

app.run()
```

```{admonition} Tipp: Navigation im 3D-Fenster
:class: note
Mit `EditorCamera()` aktivieren wir die interaktive Maus-Navigation:


- Linke Maustaste + Ziehen: Szene drehen
- Scrollrad: Zoom

- Rechte Maustaste + Ziehen: Szene verschieben

Ohne `EditorCamera()` bleibt die Kamera fest in ihrer Ausgangsposition und
die Szene ist nicht interaktiv erkundbar.
```

*Wie wird aus einer statischen Szene eine lebendige Simulation?*

Bisher haben wir Objekte nur platziert; sie bewegen sich noch nicht. Ursina
bietet dafür eine elegante Lösung: die `update()`-Funktion, die bei jedem
Bildschirmbild automatisch aufgerufen wird. In Abschnitt 8.2 werden wir
diese Funktion kennenlernen und unsere Kugel damit zum ersten Mal über die
Rampe rollen lassen.
