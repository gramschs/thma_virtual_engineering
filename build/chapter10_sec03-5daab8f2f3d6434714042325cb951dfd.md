# 10.3 VPython: Die vollständige Kugelbahn simulieren

In den letzten beiden Abschnitten haben wir die Bausteine vorbereitet:
Wegpunkte beschreiben die Geometrie der Bahn, Stoppuhr und Phyphox liefern
Messdaten zum Vergleich. Jetzt fügen wir alles zusammen. Der Euler-Loop aus
Kapitel 9 verarbeitet in diesem Abschnitt nicht mehr ein einziges Segment,
sondern wandert von Wegpunkt zu Wegpunkt durch die gesamte Bahn. Die Kugel
bewegt sich in VPython dreidimensional durch den Raum und hinterlässt dabei
eine Spur, die direkt mit dem echten Rollweg verglichen werden kann.

## Schritt 1: Geometrie und Physik vorbereiten

Wir beginnen mit der Funktion `bahn_aus_wegpunkten` aus Abschnitt 10.1 und
den Physikparametern aus Kapitel 9. Alle Parameter stehen am Anfang des
Skripts, damit sie leicht angepasst werden können.

```{code-cell} python
import numpy as np
import math

# --- Bahngeometrie aus Wegpunkten ---
def bahn_aus_wegpunkten(wegpunkte):
    laengen, winkel_rad = [], []
    for i in range(len(wegpunkte) - 1):
        delta       = wegpunkte[i+1] - wegpunkte[i]
        laenge      = np.linalg.norm(delta)
        delta_horiz = np.sqrt(delta[0]**2 + delta[2]**2)
        theta       = np.arctan2(delta[1], delta_horiz)
        laengen.append(laenge)
        winkel_rad.append(theta)
    return np.array(laengen), np.array(winkel_rad)

# Wegpunkte (aus CloudCompare exportiert oder manuell definiert)
wegpunkte = np.array([
    [0.00,  0.000, 0.00],
    [0.12, -0.020, 0.01],
    [0.25, -0.040, 0.02],
    [0.38, -0.055, 0.025],
    [0.51, -0.065, 0.030],
    [0.63, -0.080, 0.025],
    [0.76, -0.095, 0.015],
    [0.88, -0.110, 0.005],
    [1.00, -0.120, 0.000],
])

laengen, winkel_rad = bahn_aus_wegpunkten(wegpunkte)

print(f"Segmente:      {len(laengen)}")
print(f"Gesamtlänge:   {np.sum(laengen):.4f} m")
print(f"Winkelbereich: {np.degrees(winkel_rad.min()):.1f}° bis "
      f"{np.degrees(winkel_rad.max()):.1f}°")

# --- Physikparameter ---
m    = 0.1      # Masse in kg
G    = 9.81     # Erdbeschleunigung in m/s²
MU_H = 0.30     # Haftreibungskoeffizient
MU_G = 0.20     # Gleitreibungskoeffizient
dt   = 0.005    # Zeitschritt in s
```

## Schritt 2: Kräfte pro Segment berechnen

Für jedes Segment berechnen wir Normalkraft, Hangabtrieb, Haftgrenze und
Gleitreibung. Diese Werte hängen nur vom lokalen Neigungswinkel ab und sind
innerhalb eines Segments konstant.

```{code-cell} python
# Segmentweise Kräfte
F_N_seg        = m * G * np.cos(winkel_rad)
F_H_seg        = m * G * np.sin(winkel_rad)    # negativ = Gefälle = Antrieb
F_HAFT_MAX_seg = MU_H * F_N_seg
F_GLEIT_seg    = MU_G * F_N_seg

print("Kräfte pro Segment:")
print(f"{'Seg':>4} {'θ (°)':>7} {'F_H (N)':>9} {'F_Haft,max (N)':>15} {'gleitet?':>10}")
for i in range(len(laengen)):
    gleitet = abs(F_H_seg[i]) > F_HAFT_MAX_seg[i]
    print(f"{i+1:>4} {np.degrees(winkel_rad[i]):>7.2f} "
          f"{F_H_seg[i]:>9.4f} {F_HAFT_MAX_seg[i]:>15.4f} "
          f"{'ja' if gleitet else 'nein':>10}")
```

*Was passiert, wenn ein Segment so flach ist, dass die Kugel dort
nicht gleitet?*

Wenn `|F_H| ≤ F_Haft,max`, bremst die Haftreibung die Kugel bis zum
Stillstand – aber nur, wenn sie auch langsam genug ist. Eine Kugel, die bereits
mit hoher Geschwindigkeit in ein flaches Segment eintritt, wird durch die
Gleitreibung abgebremst, aber sie stoppt möglicherweise erst im nächsten
Segment. Der segmentweise Loop muss diesen Fall behandeln.

## Schritt 3: Der segmentweise Euler-Loop

Das Kernstück der Simulation ist ein doppelter Loop: der äußere Loop iteriert
über die Segmente, der innere Loop führt die Euler-Schritte innerhalb eines
Segments durch.

```{code-cell} python
import numpy as np
import math

# Parameter und Geometrie wie oben definiert (hier der Vollständigkeit halber
# wiederholt, damit die Zelle eigenständig lauffähig ist)

def bahn_aus_wegpunkten(wegpunkte):
    laengen, winkel_rad = [], []
    for i in range(len(wegpunkte) - 1):
        delta       = wegpunkte[i+1] - wegpunkte[i]
        laenge      = np.linalg.norm(delta)
        delta_horiz = np.sqrt(delta[0]**2 + delta[2]**2)
        theta       = np.arctan2(delta[1], delta_horiz)
        laengen.append(laenge)
        winkel_rad.append(theta)
    return np.array(laengen), np.array(winkel_rad)

wegpunkte = np.array([
    [0.00, 0.000, 0.00], [0.12, -0.020, 0.01],
    [0.25, -0.040, 0.02], [0.38, -0.055, 0.025],
    [0.51, -0.065, 0.030], [0.63, -0.080, 0.025],
    [0.76, -0.095, 0.015], [0.88, -0.110, 0.005],
    [1.00, -0.120, 0.000],
])
laengen, winkel_rad = bahn_aus_wegpunkten(wegpunkte)

m, G, MU_H, MU_G, dt = 0.1, 9.81, 0.30, 0.20, 0.005

# Ergebnislisten
t_verlauf = [0.0]
v_verlauf = [0.0]
s_verlauf = [0.0]   # zurückgelegte Gesamtdistanz

# Anfangszustand
v_aktuell   = 0.0
t_gesamt    = 0.0
s_gesamt    = 0.0
gleitet     = False

# Äußerer Loop: über alle Segmente
for seg_idx in range(len(laengen)):
    theta    = winkel_rad[seg_idx]
    L        = laengen[seg_idx]
    F_N      = m * G * math.cos(theta)
    F_H      = m * G * math.sin(theta)        # negativ bei Gefälle
    F_haft   = MU_H * F_N
    F_gleit  = MU_G * F_N

    # Zustandscheck beim Segmenteintritt:
    # Kugel gleitet, wenn sie schon bewegt ist ODER Hangabtrieb > Haftgrenze
    if v_aktuell > 1e-6 or abs(F_H) > F_haft:
        gleitet = True

    s_seg = 0.0   # zurückgelegte Distanz im aktuellen Segment

    # Innerer Loop: Euler-Schritte im Segment
    while s_seg < L:
        if gleitet:
            # Nettokraft: Hangabtrieb minus Gleitreibung (Reibung bremst immer)
            F_netto = -F_H - math.copysign(F_gleit, v_aktuell) if v_aktuell != 0 else -F_H - F_gleit
            # Vereinfacht für Gefälle (F_H < 0, Kugel bewegt sich in positiver Richtung):
            a = (-F_H - F_gleit) / m   # Gefälle: F_H negativ, Reibung bremst
        else:
            a = 0.0

        v_aktuell += a * dt
        if v_aktuell < 0:
            v_aktuell = 0.0   # Kugel kann nicht rückwärts rollen

        s_seg    += v_aktuell * dt
        s_gesamt += v_aktuell * dt
        t_gesamt += dt

        t_verlauf.append(t_gesamt)
        v_verlauf.append(v_aktuell)
        s_verlauf.append(s_gesamt)

print(f"Simulationsende:")
print(f"  Gesamtzeit:           {t_gesamt:.4f} s")
print(f"  Endgeschwindigkeit:   {v_aktuell:.4f} m/s")
print(f"  Gesamtstrecke:        {s_gesamt:.4f} m")
```

## Schritt 5: VPython – Die Kugel in 3D

Jetzt überführen wir die Simulation in VPython. Die Kugel bewegt sich
dreidimensional entlang der Wegpunkte, die Bahn wird als Rohrsystem aus
Zylindern dargestellt.

```{admonition} Schlüsselkonzept: Position im 3D-Raum
:class: note
Die 3D-Position der Kugel ergibt sich aus dem Startpunkt des aktuellen
Segments, der Richtung des Segments und dem bereits zurückgelegten Weg
innerhalb des Segments:

```python
richtung  = (ende - start) / laenge   # Einheitsvektor entlang des Segments
kugel.pos = vec(*start) + vec(*richtung) * s_seg
```

Dieser Ausdruck interpoliert linear zwischen Start- und Endpunkt des Segments.
```

```{code-cell} python
from vpython import *
import numpy as np
import math

# Geometrie und Parameter (wie oben)
def bahn_aus_wegpunkten(wegpunkte):
    laengen, winkel_rad = [], []
    for i in range(len(wegpunkte) - 1):
        delta       = wegpunkte[i+1] - wegpunkte[i]
        laenge      = np.linalg.norm(delta)
        delta_horiz = np.sqrt(delta[0]**2 + delta[2]**2)
        theta       = np.arctan2(delta[1], delta_horiz)
        laengen.append(laenge)
        winkel_rad.append(theta)
    return np.array(laengen), np.array(winkel_rad)

wegpunkte = np.array([
    [0.00, 0.000, 0.00], [0.12, -0.020, 0.01],
    [0.25, -0.040, 0.02], [0.38, -0.055, 0.025],
    [0.51, -0.065, 0.030], [0.63, -0.080, 0.025],
    [0.76, -0.095, 0.015], [0.88, -0.110, 0.005],
    [1.00, -0.120, 0.000],
])
laengen, winkel_rad = bahn_aus_wegpunkten(wegpunkte)

m, G, MU_H, MU_G, dt = 0.1, 9.81, 0.30, 0.20, 0.005
KUGELRADIUS = 0.015

# --- Szene ---
scene.title      = "Kugelbahn – Vollständige Simulation"
scene.background = color.white
scene.range      = 0.4
scene.width      = 1000
scene.height     = 500

# Bahn als Zylinderrohr visualisieren
for i in range(len(wegpunkte) - 1):
    start = wegpunkte[i]
    ende  = wegpunkte[i + 1]
    delta = ende - start
    cylinder(
        pos    = vec(*start),
        axis   = vec(*delta),
        radius = 0.008,
        color  = color.gray(0.6)
    )

# Wegpunkte als kleine Kugeln markieren
for wp in wegpunkte:
    sphere(pos=vec(*wp), radius=0.005, color=color.orange)

# Kugel
kugel = sphere(
    pos        = vec(*wegpunkte[0]),
    radius     = KUGELRADIUS,
    color      = color.red,
    make_trail = True,
    trail_color = color.cyan
)

# Statusanzeige
status = label(
    pos    = vec(0.5, 0.1, 0),
    text   = "Seg 0 | t=0.000s | v=0.000m/s",
    height = 12,
    color  = color.black,
    box    = False
)

# --- Simulation mit VPython-Loop ---
v_aktuell = 0.0
t_gesamt  = 0.0
gleitet   = False

for seg_idx in range(len(laengen)):
    theta   = winkel_rad[seg_idx]
    L       = laengen[seg_idx]
    start   = wegpunkte[seg_idx]
    ende    = wegpunkte[seg_idx + 1]
    F_N     = m * G * math.cos(theta)
    F_H     = m * G * math.sin(theta)
    F_haft  = MU_H * F_N
    F_gleit = MU_G * F_N

    # Richtungsvektor des Segments (Einheitsvektor)
    richtung = (ende - start) / L

    if v_aktuell > 1e-6 or abs(F_H) > F_haft:
        gleitet = True

    s_seg = 0.0

    while s_seg < L:
        rate(300)

        if gleitet:
            a = (-F_H - F_gleit) / m
        else:
            a = 0.0

        v_aktuell += a * dt
        if v_aktuell < 0:
            v_aktuell = 0.0

        s_seg    += v_aktuell * dt
        t_gesamt += dt

        # 3D-Position der Kugel aktualisieren
        kugel.pos  = vec(*start) + vec(*richtung) * min(s_seg, L)

        status.text = (f"Seg {seg_idx+1} | "
                       f"t={t_gesamt:.3f}s | "
                       f"v={v_aktuell:.3f}m/s")

print(f"Rollzeit: {t_gesamt:.4f} s")
print(f"Endgeschwindigkeit: {v_aktuell:.4f} m/s")
```

```{admonition} Checkliste: Was prüfen wir nach dem ersten Lauf?
:class: note
* **Kugel startet am richtigen Punkt:** Erster Wegpunkt ist der Startpunkt.
* **Bewegungsrichtung korrekt:** Die Kugel bewegt sich von Start nach Ende,
  nicht umgekehrt.
* **Spur sichtbar:** `make_trail = True` hinterlässt eine türkise Linie.
* **Kugel bleibt auf der Bahn:** Die Kugel bewegt sich entlang der
  Zylinder, nicht daran vorbei.
* **Simulation endet:** Der Loop stoppt nach dem letzten Segment.
* **Rollzeit plausibel:** Stimmt die Größenordnung mit der Messung überein?
```

````{admonition} Lösung
:class: tip
:class: dropdown
```python
import math
import numpy as np

def simulate(wegpunkte, laengen, winkel_rad, m, G, MU_H, MU_G, dt=0.005):
    v, t, gleitet = 0.0, 0.0, False
    for seg_idx in range(len(laengen)):
        theta   = winkel_rad[seg_idx]
        L       = laengen[seg_idx]
        F_N     = m * G * math.cos(theta)
        F_H     = m * G * math.sin(theta)
        F_haft  = MU_H * F_N
        F_gleit = MU_G * F_N
        if v > 1e-6 or abs(F_H) > F_haft:
            gleitet = True
        s = 0.0
        while s < L:
            a  = (-F_H - F_gleit) / m if gleitet else 0.0
            v += a * dt
            if v < 0:
                v = 0.0
                break
            s += v * dt
            t += dt
    return t, v

wegpunkte = np.array([
    [0.00, 0.000, 0.00], [0.12, -0.020, 0.01],
    [0.25, -0.040, 0.02], [0.38, -0.055, 0.025],
    [0.51, -0.065, 0.030], [0.63, -0.080, 0.025],
    [0.76, -0.095, 0.015], [0.88, -0.110, 0.005],
    [1.00, -0.120, 0.000],
])
laengen, winkel_rad = bahn_aus_wegpunkten(wegpunkte)
T_MESSUNG = 1.51

for mu_g in [0.10, 0.15, 0.20, 0.25, 0.30, 0.35]:
    t_sim, _ = simulate(wegpunkte, laengen, winkel_rad, 0.1, 9.81, 0.30, mu_g)
    abw = (t_sim - T_MESSUNG) / T_MESSUNG * 100
    print(f"μ_G = {mu_g:.2f}: t_sim = {t_sim:.4f} s, Δ = {abw:+.2f} %")
```

Der Reibungskoeffizient, der die Simulation dem Messwert am nächsten bringt,
ist der empirisch bestimmte Wert für dieses konkrete Material-Paar. Er weicht
häufig vom Literaturwert ab, weil Oberflächenbeschaffenheit, Temperatur und
Rollbedingungen variieren. Diese "Kalibrierung" durch Parameteranpassung ist
eine gängige Methode in der Simulationstechnik.
````

## Zusammenfassung und Ausblick

In diesem Abschnitt haben wir die vollständige segmentweise Simulation
implementiert. Die Funktion `bahn_aus_wegpunkten` liefert Längen und Winkel
aller Segmente; der doppelte Euler-Loop verarbeitet sie nacheinander und gibt
die Geschwindigkeit nahtlos von Segment zu Segment weiter. In VPython folgt
die Kugel dreidimensional dem Pfad der Wegpunkte entlang eines Zylinderrohrs.
Der Vergleich mit dem Messwert zeigt, dass der Reibungskoeffizient der
wichtigste Stellhebel für die Genauigkeit ist.

Im nächsten Abschnitt verlassen wir VPython kurz und schauen uns Ursina an:
ein Python-Framework, das das echte `.obj`-Mesh der Kugelbahn direkt laden
und als 3D-Kulisse für die Simulation verwenden kann.
