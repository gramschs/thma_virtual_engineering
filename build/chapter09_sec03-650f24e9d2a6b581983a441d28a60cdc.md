# 9.3 VPython: Kugel auf einer Rampe simulieren

In Abschnitt 9.2 haben wir das Reibungsmodell als reines Python-Skript
implementiert. Jetzt bringen wir die Visualisierung dazu: Die Kugel bewegt sich
mit korrekter Reibungsphysik, vier Kräftepfeile aktualisieren sich in Echtzeit,
und eine Statusanzeige zeigt, ob sich das Objekt gerade im Haft- oder
Gleitzustand befindet. Das Ergebnis ist eine vollständige, lehrreiche
Simulation, die wir als Vorlage für unser eigenes Objekt verwenden können.

## Was entsteht in diesem Abschnitt?

Am Ende dieses Abschnitts läuft folgender Simulator:

- Eine rote Kugel liegt auf einer grauen Rampe.
- Vier farbige Pfeile zeigen die wirkenden Kräfte: Gewichtskraft (blau),
  Normalkraft (grün), Hangabtrieb (orange) und Reibungskraft (lila).
- Solange die Kugel haftet, bleibt sie stehen und die Pfeile zeigen das
  Kräftegleichgewicht.
- Sobald der Neigungswinkel die Haftgrenze überschreitet, setzt sich die Kugel
  in Bewegung, die Kräftepfeile passen sich an und die Kugel hinterlässt eine
  Spur.
- Ein Label zeigt Simulationszeit, Geschwindigkeit und Zustand.

## Schritt 1: Szene und Parameter einrichten

```{code-cell} python
from vpython import *
import math

# Szene
scene.title      = "Kugelbahn – Simulation mit Reibung"
scene.background = color.white
scene.width      = 950
scene.height     = 500
scene.range      = 0.55

# Physikalische Parameter
NEIGUNGSWINKEL = 20.0          # Grad – hier verändern, um zu experimentieren
BAHNLAENGE     = 1.0           # Meter
KUGELRADIUS    = 0.04          # Meter
m              = 0.1           # Masse in kg
G              = 9.81          # Erdbeschleunigung in m/s²
MU_H           = 0.35          # Haftreibungskoeffizient
MU_G           = 0.25          # Gleitreibungskoeffizient
dt             = 0.005         # Zeitschritt in s

# Abgeleitete Größen
THETA      = math.radians(NEIGUNGSWINKEL)
F_N        = m * G * math.cos(THETA)
F_H        = m * G * math.sin(THETA)
F_HAFT_MAX = MU_H * F_N
F_GLEIT    = MU_G * F_N
```

## Schritt 2: Rampe und Kugel positionieren

Wir modellieren die Rampe als flachen Quader und berücksichtigen die Neigung
in den Kraftgleichungen. Die Kugel startet am linken Ende der Rampe.

```{admonition} Hinweis: Vereinfachte Visualisierung
:class: note
In dieser Simulation liegt die Rampe horizontal, die Neigung wird nur in der
Physik berücksichtigt. In Kapitel 10 werden wir das echte Mesh unserer
Kugelbahn laden und die Kugel geometrisch korrekt entlang der geneigten
Oberfläche bewegen.
```

```{code-cell} python
# Rampe
rampe = box(
    pos   = vec(0, 0, 0),
    size  = vec(BAHNLAENGE, 0.02, 0.12),
    color = color.gray(0.65)
)

# Startposition der Kugel
START_X = -(BAHNLAENGE / 2) + KUGELRADIUS
kugel   = sphere(
    pos        = vec(START_X, KUGELRADIUS + 0.01, 0),
    radius     = KUGELRADIUS,
    color      = color.red,
    make_trail = True
)

# Endposition (Abbruchbedingung)
ENDE_X = (BAHNLAENGE / 2) - KUGELRADIUS
```

## Schritt 3: Kräftepfeile erstellen

Wir erstellen vier Pfeile, die die vier Kräfte auf die Kugel darstellen. Ihre
`axis`-Eigenschaft aktualisieren wir später im Loop.

```{code-cell} python
# Skalierungsfaktor: Pfeillänge in Metern pro Newton
KRAFT_SKALA = 1.5

# Gewichtskraft: senkrecht nach unten (blau)
pfeil_FG = arrow(
    pos        = kugel.pos,
    axis       = vec(0, -F_H * KRAFT_SKALA, 0),   # Platzhalter, wird im Loop gesetzt
    shaftwidth = 0.012,
    color      = color.blue
)

# Normalkraft: senkrecht nach oben (grün)
pfeil_FN = arrow(
    pos        = kugel.pos,
    axis       = vec(0, F_N * KRAFT_SKALA, 0),
    shaftwidth = 0.012,
    color      = color.green
)

# Hangabtriebskraft: in x-Richtung (orange)
pfeil_FH = arrow(
    pos        = kugel.pos,
    axis       = vec(F_H * KRAFT_SKALA, 0, 0),
    shaftwidth = 0.012,
    color      = color.orange
)

# Reibungskraft: gegen Bewegungsrichtung (lila) – startet mit Haftreibung
pfeil_FR = arrow(
    pos        = kugel.pos,
    axis       = vec(-F_HAFT_MAX * KRAFT_SKALA, 0, 0),
    shaftwidth = 0.012,
    color      = color.purple
)
```

## Schritt 4: Legende und Statusanzeige

```{code-cell} python
    align  = "right"

```

## Schritt 5: Anfangszustand prüfen

Bevor wir den Loop starten, prüfen wir einmalig, ob das Objekt überhaupt
haftet oder sofort gleitet:

```{code-cell} python
# Anfangszustand
gleitet         = F_H > F_HAFT_MAX   # True: sofort gleiten; False: erst haften
geschwindigkeit = 0.0
t               = 0.0

# Kugelfarbe je nach Anfangszustand
kugel.color = color.red if not gleitet else color.orange

print(f"Anfangszustand: {'Gleiten' if gleitet else 'Haften'}")
print(f"F_H        = {F_H:.4f} N")
print(f"F_Haft,max = {F_HAFT_MAX:.4f} N")
```

## Schritt 6: Der vollständige Animations-Loop

Jetzt fügen wir alle Bausteine zusammen. Im Loop geschehen vier Dinge pro
Zeitschritt:

1. Zustandsübergang prüfen (Haften → Gleiten).
2. Beschleunigung berechnen.
3. Euler-Schritt ausführen.
4. Pfeile und Anzeigen aktualisieren.

```{code-cell} python
from vpython import *
import math

# --- Szene ---
scene.title      = "Kugelbahn – Simulation mit Reibung"
scene.background = color.white
scene.width      = 950
scene.height     = 500
scene.range      = 0.55

# --- Parameter ---
NEIGUNGSWINKEL = 20.0
BAHNLAENGE     = 1.0
KUGELRADIUS    = 0.04
m              = 0.1
G              = 9.81
MU_H           = 0.35
MU_G           = 0.25
dt             = 0.005
KRAFT_SKALA    = 1.5

THETA      = math.radians(NEIGUNGSWINKEL)
F_N        = m * G * math.cos(THETA)
F_H        = m * G * math.sin(THETA)
F_HAFT_MAX = MU_H * F_N
F_GLEIT    = MU_G * F_N

# --- Objekte ---
rampe = box(
    pos=vec(0, 0, 0), size=vec(BAHNLAENGE, 0.02, 0.12), color=color.gray(0.65)
)
START_X = -(BAHNLAENGE / 2) + KUGELRADIUS
ENDE_X  = (BAHNLAENGE / 2) - KUGELRADIUS

kugel = sphere(
    pos=vec(START_X, KUGELRADIUS + 0.01, 0),
    radius=KUGELRADIUS, color=color.red, make_trail=True
)

pfeil_FG = arrow(pos=kugel.pos, axis=vec(0, -F_H * KRAFT_SKALA, 0),
                 shaftwidth=0.012, color=color.blue)
pfeil_FN = arrow(pos=kugel.pos, axis=vec(0,  F_N * KRAFT_SKALA, 0),
                 shaftwidth=0.012, color=color.green)
pfeil_FH = arrow(pos=kugel.pos, axis=vec(F_H * KRAFT_SKALA, 0, 0),
                 shaftwidth=0.012, color=color.orange)
pfeil_FR = arrow(pos=kugel.pos, axis=vec(-F_HAFT_MAX * KRAFT_SKALA, 0, 0),
                 shaftwidth=0.012, color=color.purple)

status = label(
    pos=vec(ENDE_X, 0.35, 0),
    text="Zustand: Haften\nt = 0.000 s\nv = 0.000 m/s",
    height=12, color=color.black, box=False, align="right"
)

# --- Anfangszustand ---
gleitet         = F_H > F_HAFT_MAX
geschwindigkeit = 0.0
t               = 0.0
kugel.color     = color.red if not gleitet else color.orange

# --- Loop ---
while kugel.pos.x < ENDE_X:
    rate(200)

    # 1. Zustandsübergang
    if not gleitet and F_H > F_HAFT_MAX:
        gleitet     = True
        kugel.color = color.orange

    # 2. Beschleunigung
    if gleitet:
        a        = (F_H - F_GLEIT) / m
        F_R_akt  = F_GLEIT
    else:
        a        = 0.0
        F_R_akt  = F_H   # Haftreibung kompensiert Hangabtrieb genau

    # 3. Euler-Schritt
    geschwindigkeit += a * dt
    kugel.pos.x     += geschwindigkeit * dt
    t               += dt

    # 4. Pfeile und Anzeige aktualisieren
    pfeil_FG.pos = kugel.pos
    pfeil_FN.pos = kugel.pos
    pfeil_FH.pos = kugel.pos
    pfeil_FR.pos = kugel.pos

    pfeil_FG.axis = vec(0, -(m * G) * KRAFT_SKALA, 0)
    pfeil_FN.axis = vec(0,  F_N     * KRAFT_SKALA, 0)
    pfeil_FH.axis = vec(    F_H     * KRAFT_SKALA, 0, 0)
    pfeil_FR.axis = vec(   -F_R_akt * KRAFT_SKALA, 0, 0)

    zustand_text  = "Gleiten" if gleitet else "Haften"
    status.text   = (f"Zustand: {zustand_text}\n"
                     f"t = {t:.3f} s\n"
                     f"v = {geschwindigkeit:.3f} m/s")
```

```{admonition} Checkliste: Was prüfen wir nach dem ersten Lauf?
:class: note
* **Anfangszustand korrekt:** Bei θ = 20° und μ_H = 0.35 sollte die Kugel
  zunächst stehen bleiben (F_H < F_Haft,max).
* **Zustandsübergang sichtbar:** Die Kugelfarbe wechselt von Rot nach Orange,
  wenn die Haftgrenze überschritten wird – zum Beispiel nach Erhöhung des
  Winkels.
* **Pfeile bewegen sich mit:** Alle vier Pfeile folgen der Kugel und ändern
  ihre Länge, wenn sich die Reibungskraft ändert.
* **Spur sichtbar:** Die Kugel hinterlässt eine Linie, sobald sie gleitet.
* **Statusanzeige läuft:** Zeit, Geschwindigkeit und Zustand werden
  aktualisiert.
* **Loop endet:** Die Simulation stoppt, wenn die Kugel das rechte Ende
  erreicht.
```

## Schritt 7: Ergebnis experimentell erkunden

Wir verändern die Parameter, um das Modell zu verstehen. Die folgende
Mini-Übung führt durch drei gezielte Experimente.

```{admonition} Mini-Übung
:class: tip
Verändern Sie jeweils einen Parameter und beobachten Sie die Auswirkung:

1. Setzen Sie `NEIGUNGSWINKEL = 10°`. Bewegt sich die Kugel noch?
   Berechnen Sie vorher analytisch den kritischen Winkel für μ_H = 0.35.
2. Setzen Sie `MU_H = 0.15` und `MU_G = 0.10`. Wie verändert sich die
   Rollzeit? Vergleichen Sie mit dem Wert aus Abschnitt 9.2.
3. Setzen Sie `MU_H = MU_G = 0.0` (reibungsfreie Bahn). Stimmt die
   Endgeschwindigkeit mit der analytischen Formel `v = sqrt(2·g·sin(θ)·s)`
   überein?
```

````{admonition} Lösung
:class: tip
:class: dropdown
**Experiment 1:** Der kritische Winkel beträgt `arctan(0.35) ≈ 19.3°`. Bei
10° liegt der Winkel deutlich darunter – die Kugel bleibt stehen. Die
Statusanzeige zeigt dauerhaft "Haften" und `v = 0.000 m/s`.

**Experiment 2:** Mit geringerer Reibung wird die Nettokraft größer, die
Beschleunigung höher und die Rollzeit kürzer. Bei `μ_G = 0.10` und θ = 20°:
```python
import math
a = 9.81 * (math.sin(math.radians(20)) - 0.10 * math.cos(math.radians(20)))
t = math.sqrt(2 * 1.0 / a)
print(f"a = {a:.3f} m/s², t = {t:.3f} s")
# a ≈ 2.434 m/s², t ≈ 0.907 s
```

**Experiment 3:** Ohne Reibung gilt `a = g · sin(θ)`. Bei θ = 20°:
```python
import math
a     = 9.81 * math.sin(math.radians(20))
v_end = math.sqrt(2 * a * 1.0)
print(f"v_end analytisch = {v_end:.3f} m/s")
# v_end ≈ 2.590 m/s
```
Die Simulation sollte sehr gut damit übereinstimmen (Abweichung < 0.5 % bei
dt = 0.005 s).
````

## Starter-Code für das eigene Objekt

Das folgende Grundgerüst ist so gestaltet, dass es sich mit minimalen
Anpassungen auf jedes translatorische Objekt übertragen lässt. Die mit
`# ANPASSEN` markierten Stellen müssen für das eigene Objekt geändert werden:

```{code-cell} python
from vpython import *
import math

scene.title      = "Mein Objekt – Simulation"   # ANPASSEN
scene.background = color.white
scene.range      = 0.6

# Parameter – ANPASSEN
m              = 0.2          # Masse des eigenen Objekts in kg
NEIGUNGSWINKEL = 15.0         # Neigungswinkel in Grad
BAHNLAENGE     = 0.8          # Länge der Bahn in m
KUGELRADIUS    = 0.05         # Radius / halbe Höhe des Objekts in m
MU_H           = 0.30         # Haftreibungskoeffizient (Literatur oder Messung)
MU_G           = 0.20         # Gleitreibungskoeffizient
G              = 9.81
dt             = 0.005

# Kräfte
THETA      = math.radians(NEIGUNGSWINKEL)
F_N        = m * G * math.cos(THETA)
F_H        = m * G * math.sin(THETA)
F_HAFT_MAX = MU_H * F_N
F_GLEIT    = MU_G * F_N

# Objekte – ANPASSEN (Modell, Farbe, Größe)
bahn   = box(pos=vec(0, 0, 0), size=vec(BAHNLAENGE, 0.02, 0.15),
             color=color.gray(0.65))
objekt = sphere(pos=vec(-(BAHNLAENGE/2)+KUGELRADIUS, KUGELRADIUS+0.01, 0),
                radius=KUGELRADIUS, color=color.cyan, make_trail=True)

ENDE_X          = (BAHNLAENGE/2) - KUGELRADIUS
gleitet         = F_H > F_HAFT_MAX
geschwindigkeit = 0.0
t               = 0.0

while objekt.pos.x < ENDE_X:
    rate(200)
    if not gleitet and F_H > F_HAFT_MAX:
        gleitet = True
    a               = (F_H - F_GLEIT) / m if gleitet else 0.0
    geschwindigkeit += a * dt
    objekt.pos.x    += geschwindigkeit * dt
    t               += dt

print(f"Rollzeit: {t:.3f} s | Endgeschwindigkeit: {geschwindigkeit:.3f} m/s")
```
