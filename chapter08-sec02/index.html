8.2 Wie animiere ich ein Objekt?

Wir haben in Abschnitt 8.1 eine Kugel in eine 3D-Szene gesetzt. Sie sitzt dort reglos und wartet. Aber eine Simulation einer Kugelbahn, in der sich nichts bewegt, ist wenig hilfreich. Das Herzstück jeder Ursina-Simulation ist die update()-Funktion und die Zeitvariable time.dt. Mit ihnen bringen wir Bewegung in die Szene und legen das Fundament für die physikalische Simulation in Kapitel 9.

Lernziele¶

Lernziele

  • Sie können erklären, was update() ist und warum time.dt für physikalisch korrekte Animationen unverzichtbar ist.

  • Sie können eine Entity mit konstanter Geschwindigkeit gleichförmig bewegen.

  • Sie können eine einfache gleichmäßig beschleunigte Bewegung implementieren und dabei Geschwindigkeit und Position korrekt aktualisieren.

  • Sie können global-Variablen in update() korrekt verwenden.

Was passiert in jedem Frame?¶

Stellen wir uns vor, wir drehen einen Daumenkino-Film: Jede Seite zeigt ein leicht verändertes Bild, und wenn wir schnell genug blättern, sehen wir eine flüssige Bewegung. Ursina arbeitet nach demselben Prinzip. In jedem Frame berechnet die Engine den aktuellen Zustand der Szene, zeichnet ihn und ruft dann unsere update()-Funktion auf. Wir verändern dort die Positionen und Eigenschaften der Entities, und im nächsten Frame sieht die Szene entsprechend anders aus.

Warum brauchen wir time.dt und können nicht einfach eine feste Zahl addieren?

Wenn wir entity.x += 0.01 schreiben, bewegt sich das Objekt pro Frame um 0.01 m. Auf einem schnellen Rechner mit 120 fps würde die Kugel jedoch doppelt so schnell laufen wie auf einem langsamen Rechner mit 60 fps. Mit time.dt multiplizieren wir die Geschwindigkeit mit der tatsächlich verstrichenen Zeit zwischen zwei Frames; das Ergebnis ist immer gleich schnell, unabhängig vom Rechner.

Wie bewege ich ein Objekt mit konstanter Geschwindigkeit?¶

Die einfachste Animation: eine Kugel bewegt sich mit konstanter Geschwindigkeit nach rechts.

from ursina import *

app = Ursina()

# Rampe mit Länge 2 als Hintergrund
Entity(model='cube', color=color.gray,
       scale=Vec3(2.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

# Kugel
kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

# Physikalische Größe
GESCHWINDIGKEIT = 0.5   # m/s in x-Richtung

def update():
    kugel.x += GESCHWINDIGKEIT * time.dt

EditorCamera()
app.run()

Die Kugel bewegt sich mit 0.5 m/s nach rechts. Schließen wir das Fenster, wenn sie aus dem Bild verschwunden ist.

Variablen, die wir in update() nur lesen (wie GESCHWINDIGKEIT), brauchen kein global. Variablen, die wir in update() verändern wollen (wie eine Geschwindigkeit, die sich durch Beschleunigung ändert), müssen mit global deklariert werden. Das sehen wir im nächsten Beispiel.

Mini-Übung

Verändern Sie das Skript so, dass die Kugel sich schräg nach rechts oben bewegt (0.3 m/s in x-Richtung, 0.2 m/s in y-Richtung):

  1. Ersetzen Sie GESCHWINDIGKEIT = 0.5 durch GESCHWINDIGKEIT = Vec3(0.3, 0.2, 0).

  2. Ersetzen Sie in update() die Zeile kugel.x += GESCHWINDIGKEIT * time.dt durch kugel.position += GESCHWINDIGKEIT * time.dt.

Beschreiben Sie, was Sie beobachten, und erklären Sie es anhand des Koordinatensystems aus Abschnitt 8.1.

Lösung
from ursina import *

app = Ursina()

# Rampe als Hintergrund
Entity(model='cube', color=color.gray,
       scale=Vec3(2.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

# Kugel
kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

# Physikalische Größe (Geschwindigkeit in x- und y-Richtung)
GESCHWINDIGKEIT = Vec3(0.3, 0.2, 0)

def update():
    kugel.position += GESCHWINDIGKEIT * time.dt

EditorCamera()
app.run()

Die Kugel bewegt sich schräg nach rechts oben: 0.3 m/s in x-Richtung und 0.2 m/s in y-Richtung. Sie verlässt die Rampe nach oben, weil die Rampe keine physikalische Wirkung auf die Kugel hat, sondern nur ein visuelles Objekt ist. Das Physikmodell (Normalkraft, Reibung) kommt erst in Kapitel 9 dazu.

Wichtig: kugel.position += Vec3(...) * time.dt funktioniert, weil beide Seiten denselben Typ haben (Vec3). Das ursprüngliche kugel.x += Vec3(...) * time.dt würde einen Typfehler erzeugen, weil kugel.x eine einzelne Zahl erwartet.

Wie implementiere ich eine Beschleunigung?¶

Realistischer ist eine Bewegung, bei der eine Kraft die Geschwindigkeit verändert. Wir führen eine Geschwindigkeitsvariable ein, die im Euler-Cromer-Verfahren in jedem Frame aktualisiert wird: erst Geschwindigkeit, dann Position.

from ursina import *
import math

app = Ursina()

Entity(model='cube', color=color.gray,
       scale=Vec3(2.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

# Physikalische Größen
G              = 9.81
NEIGUNGSWINKEL = 15.0
A_X            = G * math.sin(math.radians(NEIGUNGSWINKEL))

geschwindigkeit = 0.0   # startet aus dem Stillstand

def update():
    global geschwindigkeit

    # Euler-Cromer-Verfahren: erst Geschwindigkeit, dann Position
    geschwindigkeit += A_X * time.dt
    kugel.x         += geschwindigkeit * time.dt

EditorCamera()
app.run()
Mini-Übung

Modifizieren Sie das Skript so, dass die Kugel bei x = 1.0 wirklich stoppt. Der korrekte Weg ist, die Abbruchbedingung vor der Integration zu prüfen:

def update():
    global geschwindigkeit

    if kugel.x >= 1.0:
        return   # sofort zurück, keine weitere Physik

    geschwindigkeit += A_X * time.dt
    kugel.x         += geschwindigkeit * time.dt

Was beobachten Sie? Warum steht die Kugel nicht exakt bei x = 1.0?

Lösung

Die Kugel bleibt knapp hinter x = 1.0 stehen. Im letzten aktiven Frame wird x noch einmal um v * time.dt erhöht, bevor die Abbruchbedingung im nächsten Frame greift. Ab dann gibt return sofort zurück und die Kugel bewegt sich wirklich nicht mehr. Das kleine Überschießen ist der unvermeidliche Diskretisierungsfehler des Euler-Cromer-Verfahrens (einer Variante der Euler-Methode): Mit kleinerer Framerate (größeres time.dt) ist der Überschuss größer, mit höherer Framerate kleiner. In Kapitel 9 werden wir diesen Effekt systematisch untersuchen.

Warum der Ansatz if kugel.x >= 1.0: geschwindigkeit = 0.0 am Ende von update() nicht funktioniert: In jedem folgenden Frame wird geschwindigkeit zunächst wieder durch += A_X * time.dt erhöht, zur Positionsänderung benutzt und erst danach auf 0 gesetzt. Die Kugel kriecht also weiter.

Wie zeige ich Text in der Szene an?¶

Mit Text können wir Informationen direkt ins Fenster einblenden:

from ursina import *
import math

app = Ursina()
window.color = color.white

Entity(model='cube', color=color.gray,
       scale=Vec3(2.0, 0.02, 0.15), position=Vec3(0, -0.05, 0))

kugel = Entity(model='sphere', color=color.red,
               scale=0.08, position=Vec3(-1.0, 0.05, 0))

anzeige = Text(
    text     = 't = 0.000 s | v = 0.000 m/s',
    position = (-0.015, 0.02),   # oder (-0.45, 0.45), etc.
    scale    = 0.1,
    color    = color.black
)

G, NEIGUNGSWINKEL = 9.81, 15.0
A_X = G * math.sin(math.radians(NEIGUNGSWINKEL))

geschwindigkeit = 0.0
t_sim           = 0.0
laeuft          = True

def update():
    global geschwindigkeit, t_sim, laeuft

    if not laeuft:
        return

    if kugel.x >= 1.0:
        laeuft = False
        
    geschwindigkeit  += A_X * time.dt
    kugel.x          += geschwindigkeit * time.dt
    t_sim            += time.dt

    anzeige.text = f't = {t_sim:.3f} s | v = {geschwindigkeit:.3f} m/s'

EditorCamera()
app.run()

In Ursina entsprechen die UI-Koordinaten einem normalisierten Bildschirmkoordinatensystem: x und y liegen typischerweise im Bereich von -0.5 bis 0.5, wobei (0, 0) die Bildschirmmitte ist. Positive x-Werte gehen nach rechts, negative nach links; positive y-Werte nach oben, negative nach unten. Elemente mit x oder y außerhalb dieses Bereichs liegen ganz oder teilweise außerhalb des sichtbaren Fensters.

Zusammenfassung und Ausblick¶

In diesem Abschnitt haben wir die update()-Funktion als Herzstück jeder Ursina-Simulation kennengelernt. Die Variable time.dt sorgt dafür, dass Animationen auf jedem Rechner gleich schnell ablaufen. Im Euler-Cromer- Verfahren aktualisieren wir erst die Geschwindigkeit, dann die Position. Mit Text zeigen wir Simulationsdaten in Echtzeit an.

Im nächsten Abschnitt setzen wir alle Bausteine zusammen: Wir bauen Schritt für Schritt eine vollständige Simulation einer Kugel auf einer geneigten Rampe auf.
